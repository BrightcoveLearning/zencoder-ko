속도 제한

API 요청 속도가 제한됩니까?

예, Zencoder는 지정된 시간 프레임 내에서 특정 메서드를 호출할 수 있는 횟수를제한합니다. 제한은 방법(리소스)별로 추적됩니다(예: GET그리고POST요청은 별도로 추적됨). 단, 출력별로 추적되는 진행 요청은 예외입니다. 예를 들어 작업 및 출력 정보를 가져 오는 호출은 모두 동일한 제한에 포함되지만 출력 진행률에 대한 호출은 출력별로 개별적으로 추적됩니다.

작업 생성 요청 (분당 1,000 개 요청으로 제한됨)과 출력 진행 요청 (출력 당 60 개 요청으로 제한됨)을 제외하고 각 계정에는 분당 60 개의 요청이 허용됩니다.

각 HTTP 응답에는X-Zencoder-Rate-Remaining "“라는 헤더가 포함되어 있습니다. 이 헤더는 현재 시간 프레임 내에서 주어진 리소스에 대해 수행 할 수있는 호출 수를 나열합니다.

할당량을 초과하면 Zencoder는 본문이 “403 Forbidden (Rate Limit Exceeded) " 인 403 오류를 반환합니다. 또한 할당량이 재설정될 때까지 시간 (초) 이 포함된 추가 헤더 " Retry-After“도 다시 전송됩니다.

API 요청 속도가 제한되는 이유는 무엇입니까?

한 고객의 Zencoder 사용이 다른 고객에게 부정적인 영향을 미치지 않도록하기 위해 API 요청을 합리적인 수로 제한합니다. 속도 제한없이 한 고객은 다른 고객이 시스템을 사용할 수 없게하는 방식으로 시스템에 요청을 넘길 수 있습니다. 이는 고객이 처음 API로 작업 할 때 우연히 주기적으로 발생합니다. 간단한 프로그래밍 오류로 인해 고객이 보내려고하지 않는 요청으로 API가 넘칠 수 있습니다.

애플리케이션이 제대로 설정되어있는 한 속도 제한은 문제가되지 않습니다. 본문에 "Rate Limit Exceeded"가 포함 된 403이 반환되면 지정된 기간이 지난 후 요청을 다시 시도하십시오.

이러한 한도를 정기적으로 실행하는 경우, 저희에게연락하여 설정 및 옵션에 대해 논의하십시오.
