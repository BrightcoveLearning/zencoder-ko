FAQ: 기본 사항

이 주제는 Zencoder에 대한 몇 가지 기본적인 질문에 대한 답변입니다.

자주하는 질문

  • 젠코더는 무엇을합니까?

    Zencoder는 온라인 비디오 변환/인코딩/트랜스코딩 서비스입니다. 웹 사이트 사용자 또는 기타 소스에서 얻은 비디오를 웹 재생, 휴대폰 또는 지원해야하는 기타 장치와 호환되는 형식으로 변환합니다.

  • Zencoder는 누구입니까?

    Zencoder는 자체 트랜스코딩 서버를 구축, 유지 관리 및 확장하는 고통스럽고 느린 프로세스를 거치지 않고 비디오를 애플리케이션에 통합하려는 개발자를 위한 것입니다. 사용자의 동영상 업로드를 허용하는 모든 웹 사이트 또는 애플리케이션에는 일종의 트랜스 코딩 백엔드가 필요합니다. 사용자가 업로드 한 동영상은 웹 및 기타 장치에서 재생할 수있는 크기와 형식이 거의 없습니다.

  • 인코딩 할 비디오를 업로드하려면 어떻게합니까?

    Zencoder의 목적은 웹 양식을 통해 각 비디오를 수동으로 업로드하지 않고도 비디오를 자동으로 변환하는 것입니다. 이를 달성하기 위해 우리는 API애플리케이션이 다음과 통신할 수 있도록 하는젠코더새 비디오를 인코딩해야 할 때 알려주십시오. 시작하려면 Zencoder가 웹을 통해 액세스할수 있는 파일 서버 또는 클라우드 호스팅 서비스 (예: Amazon S3 또는 Rackspace Cloud Files) 에 동영상이 있어야 합니다. 그런 다음 Request Builder를 사용하여 Zencoder에서 비디오를 찾을위치, 인코딩 방법 및 결과 출력 비디오를 넣을 위치를 알려주는 인코딩 작업 요청을 만들 수 있습니다. 자세한 내용은 계정의요청 빌더를참조하십시오.

  • 요청 빌더를 사용하여 업로드할 수 있는 파일 크기 제한이있습니까?

    예-크기가 5GB 미만인 파일 만 요청 빌더를 사용하여 업로드 할 수 있습니다.

  • 인코딩 작업이란 무엇입니까?

    인코딩 작업은 입력 비디오 또는 오디오 파일을 가져와 하나 이상의 출력 형식으로 변환하는 프로세스입니다. API를 통해또는 계정에서요청 빌더를사용하여 인코딩 작업을 시작할 수 있습니다. 인코딩 프로세스에서 Zencoder는먼저 호스트되는 위치에서 입력 파일을 다운로드합니다. 그런 다음 파일을 지정된 출력 설정으로 변환하고 출력 파일을 자체 서버 또는 사용자가 설정 한 특정 위치 (종종 입력 파일과 동일한 호스팅 서버)에 업로드합니다.

  • 하나의 인코딩 작업에 여러 입력을 제출할 수 있습니까?

    아니요, 작업 당 하나의 입력 만 허용됩니다.

  • Zencoder와호환되는 프로그래밍 언어는 무엇입니까?

    HTTP 요청을 보낼 수있는 모든 언어. 이 파이썬, 루비, 자바, .NET (C #, VB, F # 등), PHP, 펄, 자바 스크립트, 티클, 하스켈, 팩터, 커먼 리스프, 스몰 토크, 앞으로, ML, R, S, J, K, 배쉬, 클로쥬어, 스칼라, 모듈라 2, 모듈라 3, ColdFusion, 이맥스 리스프, 루아, C, C ++, D, Object를 포함 티브-C, 뉴턴스크립트, 매매티카, PL/SQL, 코볼, 아크, 어셈블리 여기에 선호하는 언어가 표시되지 않으면 인터넷을 지원하는지 확인하십시오. 그렇다면 아마도 괜찮을 것입니다.

    다시 말해, 젠코더 API는 들어오고 나가는 HTTP 요청일 뿐입니다. 몇 가지 통합 라이브러리를 제공하므로 좀 더 쉽게 사용할 수 있지만 라이브러리가 없으면 Zencoder와 통합하기가 그리 어렵지 않습니다. 내장 된 JSON 또는 XML 지원은 보너스이지만 JSON은 너무 간단해서 지원하지 않는 언어 (Cobol?)를 사용하는 경우 직접 해킹 할 수 있습니다.

  • 어떤 통합 라이브러리를 제공합니까?

    Zencoder와의 통합을 더쉽게 할 수 있도록 몇 가지 라이브러리를 만들었습니다.

    • 루비
    • PHP
    • 엘릭서
    • Node.js
    • 파이썬
    • 자바

    일부 고객은 자체 라이브러리를 만들어 사용할 수 있도록했습니다.

    • C#.
    • 클로저
    • 가다

    길을 따라 더 많은 라이브러리를 추가하고 싶습니다. 따라서 라이브러리를 빌드하는 경우 알려 주시면 문서에 추가하겠습니다.

  • 통합 라이브러리가 필요합니까?

    아니요.최소한 Zencoder와의통합은 JSON 또는 XML 본문을 사용하여 HTTP 요청을 보내는 것입니다. curl을사용하여 명령 줄에서이 작업을 수행 할 수있습니다. 201 Created다시 돌아오면 방금 Zencoder와 통합했습니다 .

    그 외에도 HTTP를 통해 output_notification을 수신 할 수 있습니다. URL을 알려 주시면 작업 상태를 다시 게시 해 드리겠습니다. JSON 또는 XML 본문이있는 HTTP POST를 수신하도록 설정된 사이트의 URL 만 있으면됩니다.

    마지막으로 API를 사용하여인코딩 작업으로 다른 작업 수행 , working_with_jobs, working_with_jobs 또는 working_with_jobs와 같습니다. 그것은 단지 HTTP 요청이기도합니다.

  • 통합 라이브러리는 무엇을합니까?

    통합 라이브러리는 원시 HTTP 요청에서 사용자를 추상화합니다. 사용우리의 루비 라이브러리 , 예를 들어 다음과 같은 작업을 만들 수 있습니다.

    Zencoder::Job.create({:input => 's3://bucket/key.mp4',
    :outputs => [{:label => 'vp8 for the web',
    :url => 's3://bucket/key_output.webm'}]})

    다음과 같은 작업을 수행하는 대신:

    require 'uri'
    require 'json'
    response = Net::HTTP.post_form(URI.parse('jobs.json'),
    {:input => 's3://bucket/key.mp4',
    :outputs => [{:label => 'vp8 for the web',
    :url => 's3://bucket/key_output.webm'}]}.to_json)

    라이브러리는 API로 직접 작업 할 때없는 기능을 활성화하지 않습니다. 그들은 단지 그것을 조금 더 쉽게 만듭니다.

  • 내 언어에 대한 통합 라이브러리가없는 경우 어떻게해야합니까?

    세 가지 옵션이 있습니다.

    • 나만의 빌드 및 릴리스
    • 도서관없이 작업
    • 작품에 하나가있는지 물어보십시오.

  • 자체 Zencoder라이브러리를 빌드하여 공유할 수 있습니까?

    물론이야. Zencoder라이브러리를 오픈 소스하는 경우 아마도 링크할 수도 있습니다. 원하는 언어로 Ruby 라이브러리를자유롭게 복사하십시오.

  • 내 개발 컴퓨터에서 알림을 받으려면 어떻게해야합니까?

    타사 도구 (예: ngrok) 를 사용할 수있습니다.

  • 통합 모드 란 무엇입니까?

    통합 모드는 통합을 테스트할 수 있는 계정 설정입니다. 젠코더인코딩 비용을 받지 않고 자세한 내용은테스트 작업 및 통합 모드섹션에서 확인할 수 있습니다.

  • 프라이버시 모드 란 무엇입니까?

    개인 정보 보호 모드는 특정 API 매개 변수를 적용하여 무단보기로부터 콘텐츠를 보호합니다. 우리는 또한 시스템에서 찍은 썸네일을 비활성화하고(출력 썸네일을 지정할 수 있음) 잠재적으로 민감한 정보를 난독화합니다. Zencoder직원은 어떤 이유로든 비공개 파일을 볼 수 없습니다.

    Zencoder를 통해 성인용 콘텐츠를 전송하는 경우계정이 개인 정보 보호 모드에 있어야 합니다 .

  • API 요청 속도가 제한됩니까?

    예, Zencoder는 지정된 시간 내에 특정 메서드를 호출할 수 있는 횟수를제한합니다. 제한은 출력별로 추적되는 진행 요청을 제외하고 메서드 (리소스)별로 추적됩니다. 예를 들어 작업 및 출력 정보를 가져 오는 호출은 모두 동일한 제한에 포함되지만 출력 진행률에 대한 호출은 출력별로 개별적으로 추적됩니다.

    작업 생성 요청 (분당 1,000 개 요청으로 제한됨)과 출력 진행 요청 (출력 당 60 개 요청으로 제한됨)을 제외하고 각 계정에는 분당 60 개의 요청이 허용됩니다.

    각 HTTP 응답에는X-Zencoder-Rate-Remaining "“라는 헤더가 포함되어 있습니다. 이 헤더는 현재 시간 프레임 내에서 주어진 리소스에 대해 수행 할 수있는 호출 수를 나열합니다.

    할당량을 초과하면 Zencoder는 본문이 “403 Forbidden (Rate Limit Exceeded) " 인 403 오류를 반환합니다. 또한 할당량이 재설정될 때까지 시간 (초) 이 포함된 추가 헤더 " Retry-After“도 다시 전송됩니다.

  • API 요청 속도가 제한되는 이유는 무엇입니까?

    당사는 한 고객이 Zencoder를 사용하는 것이 다른 고객에게 악영향을미치지 않도록 하기 위해 API 요청을 합리적인 숫자로 제한합니다. 속도 제한없이 한 고객은 다른 고객이 시스템을 사용할 수 없게하는 방식으로 시스템에 요청을 넘길 수 있습니다. 이는 고객이 처음 API로 작업 할 때 우연히 주기적으로 발생합니다. 간단한 프로그래밍 오류로 인해 고객이 보내려고하지 않는 요청으로 API가 넘칠 수 있습니다.

    애플리케이션이 제대로 설정되어있는 한 속도 제한은 문제가되지 않습니다. 본문에 "Rate Limit Exceeded"가 포함 된 403이 반환되면 지정된 기간이 지난 후 요청을 다시 시도하십시오.

    이러한 한도를 정기적으로 실행하는 경우, 저희에게연락하여 설정 및 옵션에 대해 논의하십시오.
