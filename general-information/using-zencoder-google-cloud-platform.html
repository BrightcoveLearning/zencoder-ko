Google Cloud Storage와 함께 Zencoder 사용

이 주제에서는 Google Cloud Platform에서 Zencoder를 사용하기위한 설정에 대해 설명합니다.

Google Cloud Storage 설정

상호 운용 가능한 GCS 자격 증명 생성

경고: Google 컴퓨팅 엔진 지역에 대한 지원은 2017년 5월 30일에 중단되었습니다.

Google Cloud Storage는 Amazon S3 API와 상호 운용 가능하지만 초기 설정 작업이 필요합니다. Zencoder에서 GCS를 시작하려면 다음을 활성화해야합니다. 상호 운용 가능한 액세스 Google Cloud Console에서 키를 생성합니다. 체크 아웃개발자 키를 생성하려면섹션이 문서상호 운용 가능한 액세스 켜기 및 액세스 키 생성에 대한 지침은

자격 증명 저장

계정에 대해 상호 운용 가능한 GCS 자격 증명 세트가 있으면 Zencoder로 안전하게 저장할 수 있습니다. Zencoder 자격 증명페이지.

자격 증명추가에서 GCP 저장소를 선택합니다 . 그런 다음 새로 생성된 자격 증명으로Google Cloud Storage Access Key ID및Secret Access Key매개 변수를 채웁니다. “gcs를 사용한 전송에 대한 기본 자격 증명”을 클릭해야 합니다. 이렇게 하면gcs://프로토콜을 지정하는 입력 또는 출력 URL이 이러한 자격 증명을 사용합니다.

트랜스 코딩

아래 예는 GCS에서 입력 비디오를 가져오고 HTTP 라이브 스트리밍 용 변환을 만들고 출력 스트림을 GCS로 보내는 작업 작업을 보여줍니다. 소유한 기존 GCS my-bucket버킷으로 변경하고요청 빌더에 붙여 넣으면 이를 테스트할 수있습니다.
