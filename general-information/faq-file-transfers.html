    자주 묻는 질문: 파일 전송

    이 항목에서는 인코딩을 위해 Zencoder로 파일을 전송하고 출력을 위해 대상으로 파일을 전송하는 방법에 대한 질문에 답합니다.

    자주하는 질문

    • 어떤 전송 프로토콜을 지원합니까? HTTP? FTP?

      입력 : HTTP, HTTPS, FTP, SFTP, Rackspace Cloud Files, Google Cloud Storage, Azure 및 Amazon S3를 지원합니다.

      출력 : FTP, SFTP, Rackspace Cloud Files, Azure, Google Cloud Storage 및 Amazon S3를 지원합니다. 현재 일반 HTTP 업로드는 지원하지 않습니다.

      Cloud Files 사용자 : 다음과 같이 프로토콜에 추가하여 사용할 지역 (DFW, ORD 또는 UK)을 지정할 수 있습니다. cf+ord://username:api_key@container/object . 현재 지원되는 클라우드 파일 지역은 DFW, ORD 및 영국뿐입니다. 기본적으로 DFW 영역이 사용됩니다.

      다른 전송 방법이 필요하십니까? 알려주세요.

    • 트랜스 코딩을 위해 원본 파일을 어디에 둘 수 있습니까?

      HTTP, HTTPS, FTP 또는 SFTP를 통해 어디서나 파일을 가져올 수 있습니다. 또한 Azure 컨테이너, Google Cloud Storage 또는 S3 버킷 (개인용 버킷 포함) 및 Rackspace Cloud Files에서 파일을 가져올 수도 있습니다.

      예:

      http://example.com/path/to/file.avi
http://s3.amazonaws.com/bucket/file.avi
s3://bucket/file.avi
gcs://bucket/file.avi
cf://username:api_key@container/file.avi
azure://account-name:account-key@container/file.avi
https://example.com/path/to/file.avi
ftp://example.com/path/to/file.avi
sftp://example.com/path/to/file.avi
      포트에 대한 참고 사항 : 파일 주소에 포트를 포함해야하는 경우 다음과 같이 URL의 도메인 끝에 선언해야합니다. 
      sftp://example.com:8888/path/to/file.avi

    • 파일을 안전하게 전송할 수 있습니까?

      확실한. SFTP 또는 HTTPS를 사용하고 다음과 같이 URL에 사용자 이름과 비밀번호를 포함합니다.

      
https://user:pass@example.com/file.mp4
sftp://user:pass@example.com/file.mp4

    • 비공개 S3, Google Cloud Storage 또는 Cloud Files 버킷에서 파일을 가져올 수 있나요?

      예. 액세스 권한을 추가 할 수 있습니다. 신임장 Zencoder 계정 또는 Zencoder 용 버킷에 대한 읽기 액세스 권한 부여 .

    • 파일 크기 제한이 있습니까?

      아니요. 파일 크기 제한이 없으며 실제로 고객이 최고의 사본을 보내도록 권장하기 위해 대용량 입력 비디오에 대해 추가 비용을 청구하지 않기로 결정했습니다. 보내 주신 동영상의 품질이 높을수록 더 높은 출력 품질을 제공 할 수 있습니다.

      그러나기간 24 시간 제한소스 비디오의.

    • 완성 된 비디오는 어디로 갑니까?

      동영상을 보낼 위치에 대한 몇 가지 옵션이 있습니다.

      파일을 FTP 또는 SFTP 서버, S3 또는 Google Cloud Storage 버킷 또는 Rackspace Cloud Files로 보낼 수 있습니다. 참조 API 문서의 URL 옵션더 많은 정보를 위해서.

      추가하는 것을 잊지 마세요신임장또는 Zencoder에게 버킷에 대한 액세스 권한 부여 S3 또는 Google Cloud Storage를 사용하는 경우.

      또는 출력 대상을 지정하지 않으면 24 시간 동안 파일을 보관합니다. 비디오를 다운로드하는 데 사용할 수있는 URL을 제공합니다. 동영상은 24 시간 후에 삭제되므로 다운로드하는 것을 잊지 마세요.

      API 버전 2부터 서버에 파일을 업로드 할 수없는 경우 대신 S3에 업로드하고 모든 파일에 대한 URL을 제공합니다.

    • 인코딩 된 파일을 S3로 보내는 경우 ACL 권한을 어떻게 설정합니까?

      API를 사용하십시오. 에 대한 API 문서를 참조하십시오. 액세스 제어더 많은 정보를 위해서.

      • 파일 전송

      Zencoder는 Amazon EC2 인스턴스에서 실행되므로 파일 및 라이브 트랜스 코딩에 사용하는 다양한 IP 주소가 있습니다. Amazon은 정기적으로 목록에 새 범위를 추가하며, 이는 AWS를 통해 찾을 수 있습니다. IP 범위 API .

      알림

      알림현재 보낸 사람sg-77f03012하지만 이는 언제든지 변경 될 수 있습니다. 이를 설명하기 위해 우리는X-Zencoder-Notification-Secret전달 된 모든 알림의 헤더. 이것은에서 찾을 수 있습니다 API대시 보드에서 현재 암호를 확인하고 필요한 경우 새 암호를 생성 할 수 있습니다.

      이 헤더는 모든 요청과 함께 전달되며 알림이 Zencoder 계정에서 전달되었음을 보장하는 데 사용할 수 있습니다. 참고: 알림 보안을 유지하려면 HTTPS 엔드 포인트에 알림을 전달해야합니다!

      참고: 서버는 새로운 데이터 센터로 이동할 수 있으며 IP 주소는 예고없이 변경 될 수 있습니다.

    • 내 비디오가있는 하드 드라이브를 Zencoder로 배송 할 수 있습니까?

      인코딩 할 비디오가 많지만 인터넷을 통해 전송하고 싶지 않은 경우 Amazon의 AWS 가져 오기 / 내보내기 . 비디오와 함께 하드 드라이브를 Amazon으로 보내면 Amazon에서 S3에 업로드합니다.

      아마존은간단한 지침드라이브 배송 준비, 배송 장소 등을 제공합니다.
