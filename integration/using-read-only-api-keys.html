개요

읽기 전용 API 요청은 계정에서 새 작업을 만들 수있는 API 키를 노출하지 않고 JSONP 요청을 할 때 유용합니다.

호환 가능한 API 요청

다음 요청은 읽기 전용 API 키와 호환됩니다.

  • 작업 세부 정보
  • 작업 진행
  • 입력 세부 정보
  • 입력 진행
  • 출력 세부 정보
  • 출력 진행

살균 예

읽기 전용 API 키를 사용하여 이루어진 요청은 민감한 정보를 보호하기 위해 응답에서 일부 정보를 제거합니다. 여기에는 URL (사용자 이름 또는 암호 정보가 제공된 경우), 특정 오류 메시지 및 기타 잠재적으로 민감한 정보가 포함되지만 이에 국한되지 않습니다.

JSONP 콜백

AJAX 및 기타 공개 요청의 경우읽기 전용 API 키는 확장자가.js 인 URL을 요청하여 JSONP를 사용하여 이 정보에 액세스할수 있습니다. 응답은 다음을 사용합니다. progress콜백을 사용하여 지정하지 않는 한callback쿼리 문자열에서. 예를 들어, 콜백이 지정되지 않은 입력 진행률에 대한 JSONP 요청은 다음과 같습니다.

progress(
  {
      "state": "processing",
      "current_event": "Downloading",
      "progress": "32.34567345"
  }
)
