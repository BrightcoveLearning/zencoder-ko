의미

지정된 주소에서 파일을 다운로드하려고했지만 파일이 없습니다. API 요청의 "입력"설정에 지정된 URL에서 다운로드를 시도합니다. 이 오류는 입력 옵션에 오타가 있거나 파일이 제거되었거나 리디렉션 루프 또는 다른 것이 있음을 의미 할 수 있습니다.

기술적 인 측면에서 이것은 몇 가지를 의미 할 수 있습니다. HTTP의 경우 이는 파일을 다운로드하려고 할 때 HTTP 응답 코드> = 300을 받았음을 의미합니다. 단, 403은 다른 오류입니다. FTP 및 FTPS를 사용하면 파일이 없거나 사용할 수 없음을 나타내는 FTP 550 응답 코드가 수신되었음을 의미합니다. 나머지 프로토콜에서는 파일을 찾을 수 없음을 의미합니다.

해결 방법

파일이 주어진 URL에 있고 우리가 연결하는 사용자가 액세스 할 수 있는지 확인하십시오. 해당 위치에없는 경우 다시 시도하기 전에 파일을 이동하십시오. 해당 위치에 존재하는 경우 파일의 권한과 지정된 사용자가 파일 경로에 액세스 할 수 있는지 확인하십시오.
