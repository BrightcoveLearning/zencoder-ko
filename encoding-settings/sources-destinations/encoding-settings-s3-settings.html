    S3 설정

    이 항목에는 S3 설정과 관련된 인코딩 설정에 대한 세부 정보가 포함되어 있습니다.

    공공의

    public:Boolean

    API 버전: V1, V2

    부모님:출력또는썸네일

    기본:그릇된

    유효한 값 :참 또는 거짓

    호환 가능한 작업 유형: 전체

    설명:

    기본적으로 S3 버킷에 저장된 파일은 공개적으로 읽을 수 없습니다. 대신 미리 준비된 액세스 정책을 사용합니다. 버킷 소유자 전체 제어 . 버킷 자체를 공개적으로 읽을 수있는 경우에도 마찬가지입니다. S3 버킷 정책은 버킷에 추가 된 새 파일에 의해 자동으로 상속되지 않습니다.

    사용공공의 S3에 대한 파일 출력을 공개적으로 읽을 수있는 것으로 플래그 지정하는 API 설정. 이는 AllUsers 그룹에 READ 권한을 부여하여 수행됩니다.

    이 설정은 출력 비디오 파일과 출력 썸네일 모두에 적용됩니다. 기본적으로 출력 파일에 설정된 사용 권한은 축소판에도 적용됩니다. 환경액세스 제어이 설정을 재정의합니다.

    수동 권한의 대안은버킷 정책을사용하는 것입니다.

    추가 정보 Amazon S3 문서에서.

    {
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "public": true,
      "thumbnails": {
        "number": 6,
        "public": false
      }
    }
  ]
}

    또한보십시오:액세스 제어과썸네일

    rrs

    rrs:Boolean

    API 버전: V1, V2

    부모의:출력

    기본:그릇된

    유효한 값 :참 또는 거짓

    호환 가능한 작업 유형: 전체

    예:진실

    설명:

    Amazon S3의 Reduced Redundancy Storage를 사용하는 경우이 플래그를 설정할 수 있으며 Zencoder는 필요한 헤더를 전달하여 S3에 RRS를 사용하도록 지시합니다.

    {
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "rrs": true
    }
  ]
}

    액세스 컨트롤

    access_control:Array

    API 버전: V1, V2

    부모님:출력또는썸네일

    유효한 값 : 피부 여자와 권한의 두 가지 설정을 포함하는 해시 배열입니다.

    호환 가능한 작업 유형: 전체

    설명:

    그만큼공공의옵션은 지정된 비디오 또는 썸네일에 대한 모든 사용자에게 S3 READ 액세스 권한을 부여합니다. 보다 세밀한 제어를 원하면액세스 제어 . 이 설정을 사용하면공공의무시할 옵션입니다. 이 설정은 출력 비디오 파일과 출력 썸네일 모두에 적용됩니다. 기본적으로 출력 파일에 설정된 사용 권한은 축소판에도 적용됩니다.

    access_control은 각각 두 가지 설정이있는 액세스 제어 권한의 배열을받습니다. 피부 여자과허가 .

    수동 권한의 대안은버킷 정책을사용하는 것입니다.

    {
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "access_control": [
        {
          "permission": "READ",
          "grantee": "someone@example.com"
        }
      ]
    }
  ]
}

    또한보십시오:공공의 , 피부 여자 , 및허가

    피부여자

    grantee:String

    API 버전: V1, V2

    부모님:출력 / 액세스 제어또는출력 / 썸네일 / 액세스 제어

    유효한 값 : 유효한 S3 피부여자 (이메일, ID 또는 URI)

    호환 가능한 작업 유형: 전체

    설명:

    S3 권한의 피부 여자 : 이메일 주소 별 사용자, CanonicalUser ID 별 사용자 또는 AWS 액세스 그룹 (http://acs.amazonaws.com/groups/global/AuthenticatedUsers 또는 http://acs.amazonaws.com/groups/) global / AllUsers). 참조 S3 ACL 문서자세한 내용은.

    의 일부로 사용액세스 제어세분화 된 S3 권한 설정.

    {
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "access_control": [
        {
          "permission": [
            "READ",
            "READ_ACP"
          ],
          "grantee": "someone@example.com"
        },
        {
          "permission": "FULL_CONTROL",
          "grantee": "cdc7931a9574b1055d5b76112021d0e9"
        },
        {
          "permission": "READ",
          "grantee": "http://acs.amazonaws.com/groups/global/AllUsers"
        },
        {
          "permission": "WRITE_ACP",
          "grantee": "http://acs.amazonaws.com/groups/global/AuthenticatedUsers"
        }
      ]
    }
  ]
}

    또한보십시오:액세스 제어과허가

    허가

    permission:String

    API 버전: V1, V2

    부모님:출력 / 액세스 제어또는출력 / 썸네일 / 액세스 제어

    유효한 값 : 다음을 포함하는 문자열 또는 문자열 배열: READ, READ_ACP, WRITE_ACP 또는 FULL_CONTROL

    호환 가능한 작업 유형: 전체

    설명:

    S3 권한 피부 여자에게 부여 할 수있는 권한입니다. 참조 S3 ACL 문서자세한 내용은.

    • READ-파일에 대한 읽기 액세스.
    • READ_ACP-주어진 파일에 대한 액세스 제어 정책을 읽을 수있는 권한.
    • WRITE_ACP-주어진 파일에 대한 액세스 제어 정책을 쓸 수있는 권한. WRITE_ACP를 가진 사용자는 파일에 대한 ACP를 변경할 수 있으므로 FULL_CONTROL과 동일합니다.
    • FULL_CONTROL-파일에 대한 READ, READ_ACP 및 WRITE_ACP 권한을 부여합니다.

    의 일부로 사용액세스 제어세분화 된 S3 권한 설정.

    {
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "access_control": [
        {
          "permission": [
            "READ",
            "READ_ACP"
          ],
          "grantee": "someone@example.com"
        },
        {
          "permission": "FULL_CONTROL",
          "grantee": "cdc7931a9574b1055d5b76112021d0e9"
        },
        {
          "permission": "READ_ACP",
          "grantee": "http://acs.amazonaws.com/groups/global/AllUsers"
        },
        {
          "permission": "WRITE_ACP",
          "grantee": "http://acs.amazonaws.com/groups/global/AuthenticatedUsers"
        }
      ]
    }
  ]
}
