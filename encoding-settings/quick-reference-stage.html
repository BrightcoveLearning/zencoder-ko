인코딩 설정

이 항목에서는 각 설정에 대한 자세한 가이드에 대한 링크와 함께 모든 인코딩 설정에 대한 표를 제공합니다.

일

환경 기본 기술
api_key 없음 Zencoder 계정의 API 키입니다.
dynamic_profile_options 없음 CAE를 사용하면 이러한 출력의 출력 수와 매개 변수가 각 입력 파일에 대해 동적으로 변경 될 수 있습니다. dynamic_profile_options 설정은 변환을 만드는 방법을 정의합니다.
generate_dynamic_profile 그릇된 CAE를 사용하면 이러한 출력의 출력 수와 매개 변수가 각 입력 파일에 대해 동적으로 변경 될 수 있습니다.
input 없음 트랜스 코딩 할 파일을 다운로드 할 수있는 HTTP, S3, Cloud Files, GCS, FTP, FTPS, SFTP, Azure 또는 Aspera URL.
outputs 없음 출력 설정의 배열 또는 해시입니다.
region 우리 파일이 처리되는 리전: 미국, 유럽, 아시아 또는 호주.
test 그릇된 작업에 대해 테스트 모드 (“통합 모드”) 를 활성화합니다.
private 그릇된 작업에 대한 개인 정보 보호 모드를 활성화합니다.
download_connections 5 다운로드 가속화를 위해 여러 동시 연결을 활용합니다 (일부 상황에서).
pass_through 없음 이 작업과 함께 저장할 선택적 정보입니다.
mock 그릇된 모의 작업 요청을 보냅니다.
grouping 없음 이 작업에 대한 보고서 그룹입니다.
aspera_transfer_policy 공정한 Aspera 파일 전송에 사용 가능한 대역폭을 할당하는 방법.
transfer_minimum_rate 1000 최소 데이터 전송 속도 (Kbps)입니다.
transfer_maximum_rate 250000 데이터 전송 최대 값에 대한 목표 속도 (Kbps)입니다.
expected_md5_checksum 없음 입력 파일의 예상 체크섬입니다.
credentials 없음 별명으로 저장된 자격 증명을 참조합니다.

일반 출력 설정

환경 기본 기술
type 표준 출력 할 파일 유형입니다.
label 없음 이 출력에 대한 선택적 레이블입니다.
dynamic_profile_rendition 없음 dynamic_profile_rendition 설정은 이 출력에서 동적으로 생성된 CAE 프로파일의 설정과 특히 설정을 사용할 프로파일의 변환을 사용하도록 지정합니다.
url 없음 트랜스 코딩 된 파일을 넣을 S3, Cloud Files, GCS, FTP, FTPS, SFTP, Aspera, Azure, HTTP 또는 RTMP URL.
secondary_url 없음 트랜스 코딩 된 파일을 넣을 S3, Cloud Files, GCS, FTP, FTPS, SFTP, HTTP, RTMP, Aspera 또는 Azure URL.
base_url 없음 파일 이름없이 트랜스 코딩 된 파일을 넣을 기본 S3, Cloud Files, GCS, FTP, FTPS, SFTP, Azure 또는 Aspera 디렉터리 URL.
filename 없음 완성 된 파일의 파일 이름입니다.
package_filename 없음 패키지화 된 출력의 파일 이름입니다.
package_format 없음 출력 파일에 사용할 Zip / 패키징 형식입니다.
device_profile 없음 모바일 장치 호환성에 사용할 장치 프로필입니다.
strict 그릇된 엄격 모드를 활성화합니다.
skip_video 그릇된 비디오 트랙을 출력하지 마십시오.
skip_audio 그릇된 오디오 트랙을 출력하지 마십시오.
source 없음 다른 작업의 레이블을 참조하고 입력 파일 대신 처리를 위해 해당 출력에서 만든 비디오를 사용합니다.
credentials 없음 별명으로 저장된 자격 증명을 참조합니다.
generate_md5_checksum 그릇된 출력 파일의 MD5 체크섬을 생성합니다.
parallel_upload_limit S3의 경우 30, 기타 모든 대상의 경우 10. 시도 할 최대 동시 업로드 수입니다.
headers 없음 파일을 업로드 할 때 파일과 함께 보낼 HTTP 헤더입니다.

CAE 설정

환경 기본 기술
min_renditions 1 생성 할 최소 변환 수입니다.
The maximum number of renditions to generate 8 생성 할 최대 변환 수입니다.
min_resolution 192x108 래더에 대해 고려할 최소 해상도입니다.
max_resolution 소스 해상도 래더에 대해 고려할 최대 해상도입니다.
max_frame_rate 30 사용할 수있는 최대 프레임 속도입니다.
keyframe_rate 0.5 키 프레임 속도. 이 값은 0.5 또는 1 일 수 있습니다. SSAI 사용 콘텐츠의 경우이 값은 1이어야합니다.
fixed_frame_rate 진실 생성기에 모든 변환에서 동일한 프레임 속도를 사용하도록 지시합니다. 거짓 인 경우 360p 이하의 해상도에서 H.264 기준 프로필 변환에 프레임 속도가 절반으로 할당됩니다.
min_bitrate 200 사용할 수있는 최소 비트 전송률 [kbps]입니다.
max_bitrate 15000 사용할 수있는 최대 비트 전송률 [kbps]입니다.
max_first_rendition_bitrate 400 첫 번째 변환에 사용할 수있는 최대 비트 전송률 [kbps]입니다.
min_granularity 30 변환 간 비트 전송률의 최소 상대 변경.
max_granularity 90 변환 간 비트 전송률의 최대 상대 변경.
video_codec h264 사용할 비디오 코덱입니다.
bitrate_cap_to_bitrate_ratio 1.3 VBR 인코딩에 사용할 평균 비트 전송률에 대한 최대 비트 전송률의 비율입니다.
buffer_size_to_bitrate_ratio 2.0 VBR 인코딩에 사용할 평균 비트 전송률에 대한 디코더 버퍼 크기의 비율입니다.
bitrate_cap_offset 64 최대 비트 전송률을 계산할 때 상수가 추가됩니다.
buffer_size_offset 85 VBV 버퍼 크기를 계산할 때 추가되는 상수입니다.
quality_rate_tune 0 품질 대 속도 조정 매개 변수.
select_baseline_profile_configuration 진실 프로필에 사용 된 변환 중 하나 이상이 기준 프로필이됩니다.
video_configurations 없음 CAE 인코딩의 일부로 우선적으로 또는 의무적으로 생성되어야하는 해상도, 프레임 속도 및 관련 코덱 설정을 설명하는 구조의 배열입니다.

형식 및 코덱

환경 기본 기술
format 출력 파일 이름과 비디오 또는 오디오 코덱에 의해 결정됩니다. 그렇지 않으면: mp4 (표준 출력의 경우); ts (세그먼트 출력의 경우). 사용할 출력 형식입니다.
video_codec 형식, 프로파일 또는audio_codec h264 (제공되지 않은 경우) 에 의해 결정됩니다. 사용할 비디오 코덱입니다.
audio_codec 형식, 프로파일 또는video_codec제공된 파일이 없는 경우 aac로 결정됩니다. 사용할 오디오 코덱입니다.

해결

환경 기본 기술
size 없음 출력 비디오의 해상도 (WxH, 픽셀)입니다.
width 없음 출력 비디오의 최대 너비 (픽셀)입니다.
height 없음 출력 비디오의 최대 높이 (픽셀)입니다.
upscale 그릇된 입력이 목표 출력 해상도보다 작은 경우 출력을 업 스케일링합니다.
aspect_mode 보존하다 입력 파일의 가로 세로 비율이 대상 가로 / 세로 가로 세로 비율과 일치하지 않을 때 수행 할 작업
sample_aspect_ratio 없음 출력 비디오에 사용할 샘플 (픽셀) 종횡비입니다. 크기, 너비, 높이 및aspect_mode모드 옵션은 모두 디스플레이 크기가 아니라 인코딩된 픽셀 크기를 나타냅니다.
force_interlace_headers 그릇된 일부 디코더는 인터레이스 콘텐츠를 요청하지만 프로그레시브 콘텐츠는 정상적으로 재생됩니다. 이 설정은 프로그레시브 콘텐츠의 메타 데이터에 "인터레이스"헤더를 추가하여 실제로 인터레이스 된 것처럼 보이게합니다.

속도 제어

환경 기본 기술
quality 3 대상 화질과 일치하도록 최적의 비디오 비트 전송률을 자동 선택합니다.
video_bitrate 없음 대상 비디오 비트 전송률 (kbps)입니다. 특정 비트 전송률을 대상으로하지 않는 한 품질 설정을 선택하는 경우 필요하지 않습니다.
audio_quality 3 대상 음질과 일치하도록 최상의 오디오 비트 전송률을 자동 선택합니다.
audio_bitrate 없음 대상 오디오 비트 전송률 (kbps)입니다. 특정 비트레이트를 타깃팅하려는 경우가 아니면audio_quality설정을 선택하는 경우에는 필요하지 않습니다.
max_video_bitrate 없음 최대 평균 비트 전송률입니다.
speed 3 목표 트랜스 코딩 속도입니다. 느린 인코딩은 일반적으로 더 고급 압축을 허용합니다.
decoder_bitrate_cap 없음 디코더 버퍼에 공급되는 최대 비트 전송률입니다. 일반적으로 스트리밍을 위한 비디오 또는 특정 디바이스를 타겟팅하는 데 사용됩니다 (예: Blu-Ray ).
decoder_buffer_size 없음 디코더 버퍼의 크기로, 와 함께 사용됩니다bitrate_cap .
one_pass 그릇된 원 패스 인코딩을 강제합니다.
audio_constant_bitrate 그릇된 가능하면 오디오에 대해 고정 비트 레이트 모드를 활성화하십시오.

프레임 속도

환경 기본 기술
frame_rate 원래 (입력) 프레임 속도입니다. 사용할 프레임 속도입니다.
max_frame_rate 없음 사용할 최대 프레임 속도입니다.
decimate 없음 입력 프레임 속도를 제수로 줄입니다.
keyframe_interval 250 각 키 프레임 사이의 최대 프레임 수입니다. H.264 비디오의 경우 키 프레임은 IDR 프레임입니다.
keyframe_rate keyframe_interval 250의 초당 키 프레임 수입니다.
fixed_keyframe_interval 그릇된 고정 키 프레임 간격 모드를 활성화합니다 (VP6, VP8, VP9 및 H.264 만 해당).
forced_keyframe_interval 없음 지정된 간격으로 키 프레임을 적용합니다 (H.264 만 해당).
forced_keyframe_rate 없음 프레임 속도를 고려하여 초당 키 프레임 수를 지정합니다 (H.264 만 해당).
generate_frame_index 그릇된 비디오 프레임 인덱스 파일을 생성합니다.
frame_index_filename 없음 generate_frame_index가 활성화된 경우 프레임 인덱스의 기본 파일 이름을 재정의합니다.

비디오

환경 기본 기술
video_reference_frames 3 사용할 참조 프레임 수 (H.264, HEVC).
video_codec_profile H.264: 기준선, HEVC: 메인, VP9: 0 사용할 비디오 코덱 프로필 (H.264, HEVC, VP9).
video_codec_level 출력 영상을 기반으로 레벨 차트를 사용하여 자동 계산됩니다. 참조H.264-levels , HEVC-levels 사용할 비디오 코덱 수준 (H.264, HEVC)입니다.
video_bframes 0 연속 B 프레임의 최대 수 (H.264, HEVC)
color_metadata 보존하다 출력 비디오의 입력에서 색상 메타 데이터 정보를 유지하거나 삭제합니다.

오디오

환경 기본 기술
audio_sample_rate 없음 Hz 단위의 오디오 샘플 속도입니다.
max_audio_sample_rate 없음 최대 오디오 샘플 속도 (Hz)입니다.
audio_channels 원본 파일이 모노 인 경우 1; 그렇지 않으면 2. 오디오 채널 수: 1 또는 2.
audio_bit_depth 16 샘플당 비트 수: 16, 24, 32.
audio_language 없음 오디오 트랙의 언어를 설명합니다.
input_audio_channels 없음 입력 오디오 트랙-채널 레이아웃을 설정합니다.
output_audio_channels 없음 출력 오디오 채널 레이아웃을 설정합니다.

썸네일

환경 기본 기술
thumbnails 없음 주어진 비디오의 썸네일을 캡처합니다.
label 없음 각 썸네일 그룹 집합을 식별하는 레이블입니다.
format png 미리보기 이미지의 형식입니다.
number 없음 균등 한 간격의 여러 축소판.
start_at_first_frame 그릇된 첫 번째 프레임부터 썸네일 생성을 시작합니다.
interval 없음 균등 한 간격 (초)으로 미리보기 이미지를 찍습니다.
interval_in_frames 없음 프레임 단위로 균등 한 간격으로 썸네일을 찍습니다.
times 없음 썸네일을 가져올 시간 (초)의 배열입니다.
aspect_mode 보존하다 입력 파일의 가로 세로 비율과 다른 축소판 너비 / 높이를 처리하는 방법입니다.
size 없음 축소판 해상도는 로WxH .
width 없음 축소판의 최대 너비 (픽셀)입니다.
height 없음 축소판의 최대 높이 (픽셀)입니다.
base_url 없음 파일 이름없이 썸네일을 배치 할 기본 S3, Cloud Files, GCS, FTP, FTPS 또는 SFTP 디렉토리 URL.
prefix 틀 축소판 파일 이름의 접두사입니다.
filename 틀 보간 된 썸네일 파일 이름.
public 그릇된 S3에서 출력을 공개적으로 읽을 수 있도록합니다.
access_control 없음 S3로 전송되는 파일에 대한 세분화 된 액세스 제어 규칙.
grantee 없음 세분화 된 S3 액세스 제어 권한에 대한 피부 여자를 설정합니다.
permission 없음 세분화 된 액세스 제어를 사용할 때 피부 여자에 대한 권한을 설정합니다.
rrs 그릇된 Amazon S3의 감소 된 중복 스토리지.
headers 없음 HTTP headers미리보기 이미지를 업로드할 때 미리보기 이미지와 함께 보낼 수 있습니다.
credentials 없음 별명으로 저장된 자격 증명을 참조합니다.
parallel_upload_limit S3의 경우 30, 기타 모든 대상의 경우 10. 시도 할 최대 동시 업로드 수입니다.

워터 마크

환경 기본 기술
watermarks 없음 출력 비디오에 하나 이상의 워터 마크를 추가합니다.
url 없음 워터 마크로 사용할 원격 이미지 파일의 URL입니다.
x -10 x축에 워터마크를 배치할 위치입니다.
