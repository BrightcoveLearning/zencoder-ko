audio_gain

audio_gain:Float

API 버전: V1, V2

상위: 출력

유효한 값: -60에서 60 사이의 숫자

호환 가능한 작업 유형: 모두

예: 5

기술:

오디오에 게인 양을 적용합니다. 다른 효과를 적용한 후 지정된 양만큼 오디오 볼륨을 높이거나 낮 춥니 다. 다른 효과를 적용한 후 레벨을 조정하거나 레벨을 이미 알고있는 파일을 조정하는 데 주로 유용합니다. 최대 볼륨을 약간 초과하는 사운드는 충실도를 유지하기 위해 부드럽게 제한되지만 최대 볼륨 이상으로 크게 증가하면 사운드가 왜곡됩니다. 왜곡 없이 볼륨을 자동으로 늘리려면 audio_normalize를참조하십시오. 최대 60dB의 양 또는 음의 dB로 지정됩니다.

{
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "audio_gain": 5
    }
  ]
}

참조: 오디오_정규화

audio_normalize

audio_normalize:Boolean

API 버전: V1, V2

상위: 출력

기본값:거짓

유효한 값:참 또는 거짓

호환되는 작업 유형: vod

예: true

기술:

오디오를 0dB로 표준화합니다. 이것은 왜곡을 일으키지 않고 가능한 한 오디오의 볼륨을 증가시킵니다. 확장 및 압축 효과로 일관된 조정을 보장하기 위해 해당 효과 전후의 볼륨을 정규화합니다. 이러한 지점 중 하나에서만정규화하려면 audio_pre_정규화및 audio_post_정규화 를 참조하십시오. audio_normalize는 audio_pre_정규화와 audio_post_정규화를 모두 지정하는 것과 같습니다.

{
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "audio_normalize": true
    }
  ]
}

참조: 오디오_게인 , 오디오_프리_정규화및오디오_포스트_정규화

audio_highpass

audio_highpass:Integer

API 버전: V1, V2

상위: 출력

유효한 값: Hz 단위의 주파수, 5 ~ 24000

호환 가능한 작업 유형: 모두

예: 10

기술:

지정된 주파수 (Hz)에서 오디오에 고역 통과 필터를 적용합니다. 이렇게하면 지정된 임계 값 미만의 오디오 주파수가 출력에 포함되지 않습니다. 이 설정의 매우 일반적인 용도는 녹음 중에 외부에서 운전하는 트럭의 소리 나 바람이나 마이크에 부딪히는 소리와 같은 오디오의 럼블 / 노이즈를 줄이는 것입니다. DC-Shift는 또한 10Hz와 같은 저주파에서 고역 통과 필터를 지정하여 감소 / 제거 할 수 있습니다.

{
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "audio_highpass": 10
    }
  ]
}

참조: 오디오_저역 통과

    {
      "audio_lowpass": 3000
    }
  ]
}

audio_compression_ratio

audio_compression_ratio:Float

API 버전: V1, V2

상위: 출력

유효한 값: 1.0에서 30.0

호환 가능한 작업 유형: 모두

예: 2.5

기술:

지정된 audio_compression_threshold 이상으로 지정된 N:1 비율로 볼륨을 줄여 오디오의 동적 범위를압축합니다. 1.0에서 30.0 사이의 비율이 유효합니다. 압축은 기본적으로 조용한 지점과 큰 지점 사이에서 오디오 볼륨을보다 일관되게 만드는 데 유용합니다. 이것은 시끄러운 부분 (고함, 꽝 닫은 문 등)을 덜 울리는 데 사용할 수 있습니다. audio_gain또는 audio_정규화와 함께 사용하면 음성의 조용한 부분을 더 청취하게 만들 수 있습니다.

{
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "audio_compression_ratio": 2.5
    }
  ]
}

참조: 오디오_압축_임계값
