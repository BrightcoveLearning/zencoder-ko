    오디오 처리 설정

    이 항목에는 오디오 처리와 관련된 인코딩 설정에 대한 세부 정보가 포함되어 있습니다. 기타 오디오 관련 설정은오디오과속도 제어 .

    오디오 게인

    audio_gain:Float

    API 버전: V1, V2

    부모의:출력

    유효한 값 : -60에서 60 사이의 숫자

    호환 가능한 작업 유형: 전체

    예: 5

    설명:

    오디오에 게인 양을 적용합니다. 다른 효과를 적용한 후 지정된 양만큼 오디오 볼륨을 높이거나 낮 춥니 다. 다른 효과를 적용한 후 레벨을 조정하거나 레벨을 이미 알고있는 파일을 조정하는 데 주로 유용합니다. 최대 볼륨을 약간 초과하는 사운드는 충실도를 유지하기 위해 부드럽게 제한되지만 최대 볼륨 이상으로 크게 증가하면 사운드가 왜곡됩니다. 보다 audio_normalize왜곡없이 자동으로 볼륨을 증가시킵니다. 최대 60dB의 양 또는 음의 dB로 지정됩니다.

    {
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "audio_gain": 5
    }
  ]
}

    또한보십시오: audio_normalize

    오디오 정규화

    audio_normalize:Boolean

    API 버전: V1, V2

    부모의:출력

    기본:그릇된

    유효한 값 :참 또는 거짓

    호환 작업 유형: VOD

    예:진실

    설명:

    오디오를 0dB로 정규화합니다. 이것은 왜곡을 일으키지 않고 가능한 한 오디오의 볼륨을 증가시킵니다. 확장 및 압축 효과로 일관된 조정을 보장하기 위해 해당 효과 전후의 볼륨을 정규화합니다. 보다 audio_pre_normalize과 audio_post_normalize해당 지점 중 하나에서만 정규화합니다. audio_normalize는 audio_pre_normalize 및 audio_post_normalize를 모두 지정하는 것과 같습니다.

    {
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "audio_normalize": true
    }
  ]
}

    또한보십시오: audio_gain , audio_pre_normalize , 및 audio_post_normalize

    오디오_하이패스

    audio_highpass:Integer

    API 버전: V1, V2

    부모의:출력

    유효한 값 : 5 ~ 24000에서 Hz의 주파수

    호환 가능한 작업 유형: 전체

    예: 10

    설명:

    지정된 주파수 (Hz)에서 오디오에 고역 통과 필터를 적용합니다. 이렇게하면 지정된 임계 값 미만의 오디오 주파수가 출력에 포함되지 않습니다. 이 설정의 매우 일반적인 용도는 녹음 중에 외부에서 운전하는 트럭의 소리 나 바람이나 마이크에 부딪히는 소리와 같은 오디오의 럼블 / 노이즈를 줄이는 것입니다. DC-Shift는 또한 10Hz와 같은 저주파에서 고역 통과 필터를 지정하여 감소 / 제거 할 수 있습니다.

    {
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "audio_highpass": 10
    }
  ]
}

    또한보십시오: audio_lowpass

    오디오_로우 패스

    audio_lowpass:Integer

    API 버전: V1, V2

    부모의:출력

    {
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "audio_lowpass": 3000
    }
  ]
}

    또한보십시오: audio_highpass

    오디오_압축_비율

    audio_compression_ratio:Float

    API 버전: V1, V2

    부모의:출력

    유효한 값 : 1.0에서 30.0 에

    호환 가능한 작업 유형: 전체

    예: 2.5

    설명:

    지정된 비율 인 N : 1 이상으로 볼륨을 줄여 오디오의 동적 범위를 압축합니다. audio_compression_threshold . 1.0에서 30.0 사이의 비율이 유효합니다. 압축은 기본적으로 조용한 지점과 큰 지점 사이에서 오디오 볼륨을보다 일관되게 만드는 데 유용합니다. 이것은 시끄러운 부분 (고함, 꽝 닫은 문 등)을 덜 울리는 데 사용할 수 있습니다. 와 함께 사용 audio_gain또는 audio_normalization을 사용하면 음성의 조용한 부분을 더 잘들을 수 있습니다.

    {
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "audio_compression_ratio": 2.5
    }
  ]
}

    또한보십시오: audio_compression_threshold

    오디오 압축_임계값

    audio_compression_threshold:Float

    API 버전: V1, V2

    부모의:출력

    기본: -20

    유효한 값 : 0에서 -120 사이의 음수

    호환 가능한 작업 유형: 전체

    예: -30

    설명:

    다음과 같은 경우 오디오 압축이 적용되는 임계 값을 설정합니다. audio_compression_ratio지정됩니다. 값은 -120까지 음수로 dB 단위로 지정됩니다.

    {
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "audio_compression_threshold": -30
    }
  ]
}

    또한보십시오: audio_compression_ratio

    오디오 확장_임계값

    audio_expansion_threshold:Float

    API 버전: V1, V2

    부모의:출력

    기본: -35

    유효한 값 : 0에서 -120 사이의 음수

    호환 가능한 작업 유형: 전체

    예: -10

    설명:

    다음과 같은 경우 오디오 압축이 적용되는 임계 값을 아래로 설정합니다. audio_expansion_ratio지정됩니다. 값은 -120까지 음수로 dB 단위로 지정됩니다.

    {
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "audio_expansion_threshold": -10
    }
  ]
}

    또한보십시오: audio_expansion_ratio

    API 버전: V1, V2

    부모의:출력

    {

  "outputs": [

    }
  ]
}
