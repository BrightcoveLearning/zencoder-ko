개요

Zencoder는 방송 및 전문 워크 플로를위한 고급 인코딩 형식과 코덱을 지원합니다.

video_codec_preset

video_codec_preset:String

API 버전: V2

부모: outputs

유효한 값: avci_50, avci_100

호환 가능한 작업 유형: 모두

예: avci_100

기술:

특정 산업 표준과의 호환성을 허용하는 비디오 인코딩 사전 설정.

표준:

  • avci_50은 AVC 10 비트 프레임 내 전용 압축입니다. 50Mbit / s, High 10 Intra Profile, 레벨 3.2, 4. SMTP RP 2027:2007을 참조하십시오.
  • avci_100은 AVC 10 비트 프레임 내 전용 압축입니다. 100 베가비트/초, 높음 4:2:2 인트라 프로파일, 레벨 4.1. SMTP RP 2027:2007을 참조하십시오.
{
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "video_codec_preset": "avci_100",
      "video_bit_depth": 10
    }
  ]
}

format_preset

format_preset:String

API 버전: V2

부모: outputs

유효한 값: ts_dvb, ts_케이블 랩, ts_브로드캐스트, as11_hd

호환 가능한 작업 유형: 모두

예: ts_broadcast

기술:

특정 산업 표준과의 호환성을 허용하는 형식 사전 설정. 2 개의 오디오 채널 만 지원됩니다.

표준:

  • ts_broadcast는 표준 MPEG-TS 브로드 캐스트 호환 스트림을 생성합니다.
  • ts_dvb는 DVB (Digital Video Broadcasting) 규격 스트림을 생성합니다.
  • ts_cablelabs는 CableLabs 호환 MPEG-TS 스트림을 생성합니다.
  • as11_hd는 MXF-OP1A 컨테이너에있는 AVC-Intra 100 비디오, 4 또는 16 채널의 PCM 오디오입니다.
{
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "format_preset": "ts_broadcast"
    }
  ]
}

참조: 형식

ts_muxrate

ts_muxrate:Integer

API 버전: V2

부모: outputs

유효한 값: 양의 정수. 최대 1000000.

호환 가능한 작업 유형: 모두

예: 2600

기술:

전송 스트림을 다중화 할 수있는 최대 속도 (kbps)입니다. ts 형식에만 사용됩니다.

{
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "format_preset": "ts_broadcast",
      "ts_muxrate": 2600,
      "video_bitrate": 2000,
      "audio_bitrate": 128,
      "audio_sample_rate": 48000,
      "audio_constant_bitrate": true
    }
  ]
}

참조: format_preset

ts_cbr

ts_cbr:Boolean

기본값:거짓

유효한 값:참 또는 거짓

  "outputs": [
    {
      "format_preset": "ts_broadcast",
      "ts_cbr": true,
      "video_bitrate": 2000,
      "audio_bitrate": 128,
      "audio_sample_rate": 48000,
      "audio_constant_bitrate": true
    }
  ]
}

참조: format_preset
