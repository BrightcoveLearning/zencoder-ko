audio_sample_rate

audio_sample_rate:Integer

API 버전: V1, V2

부모: outputs

유효한 값: 유효한 샘플링 속도. 코덱에 따라 다릅니다. 일반적으로 8000, 11025, 12000, 16000, 22050, 32000, 44100, 48000, 96000

호환되는 작업 유형: vod

예: 44100

기술:

Hz 단위의 오디오 샘플 속도입니다.

정당한 이유가있는 경우에만이 설정을 사용하는 것이 좋습니다. 출력 샘플 레이트를 강제로 하면 예기치 않은 문제가 발생할 수 있습니다. 예를 들어, 각 audio_codec 은 제한된 범위의 샘플레이트만 지원하며 각 샘플 속도는 제한된 비트 전송률 범위와만 호환됩니다. 16kbps에서 48Khz AAC 또는 320kbps에서 8000hz MP3를 인코딩 할 수 없습니다.

 

기본적으로 입력 샘플 속도가 사용되지만 (즉, 오디오가 리샘플링되지 않음) 96000보다 높은 샘플 속도는 최대 96000으로 감소됩니다.

FLV 비디오형식은 11025, 22050 및 44100의 샘플속도만 지원합니다. 기본적으로 FLV 비디오에만 사용하고 다른 샘플 속도를 FLV와 호환되는 가장 가까운 속도로 이동합니다. 호환되지 않는 샘플 레이트로 인해 유효한 sample_rate로 자동 수정하지 않고 작업이 실패하도록 하려면strict_mode활성화합니다.

  {
    "input": "s3://zencodertesting/test.mov",
    "outputs": [
      {
        "audio_sample_rate": 44100
      }
    ]
  }

참고: 오디오_비트레이트및오디오_품질

max_audio_sample_rate

max_audio_sample_rate:Integer

API 버전: V1, V2

부모: outputs

유효한 값: 유효한 샘플링 속도. 코덱에 따라 다릅니다. 일반적으로 8000, 11025, 12000, 16000, 22050, 32000, 44100, 48000, 96000

호환 가능한 작업 유형:라이브

예: 44100

기술:

최대 오디오 샘플 속도입니다. 샘플 속도가 제공된 수를 초과하지 않도록 audio_sample_rate설정을 재정의합니다.

입력 비디오의 샘플 레이트가max_audio_sample_rate 48000이고 22050으로 설정된 경우 오디오를 22050으로 리샘플링합니다. 샘플 레이트가 11025이면 그대로 둡니다.

대상 재생 장치가 더 높은 샘플 속도를 지원하지 않는 경우이 옵션을 사용합니다.

기본적으로 입력 샘플 속도가 사용되지만 (즉, 오디오가 리샘플링되지 않음) 96000보다 높은 샘플 속도는 최대 96000으로 감소됩니다.

FLV 비디오형식은 11025, 22050 및 44100의 샘플속도만 지원합니다. 기본적으로 FLV 비디오에만 사용하고 다른 샘플 속도를 FLV와 호환되는 가장 가까운 속도로 이동합니다. 호환되지 않는 샘플 레이트로 인해 유효한 sample_rate로 자동 수정하지 않고 작업이 실패하도록 하려면strict_mode활성화합니다.

  {
    "input": "s3://zencodertesting/test.mov",
    "outputs": [
      {
        "max_audio_sample_rate": 44100
      }
    ]
  }

참조: 오디오_샘플_레이트 , 오디오_비트레이트및오디오_품질

audio_channels

audio_channels:Integer

API 버전: V1, V2

부모: outputs

기본: 원본 파일이 모노 인 경우 1; 그렇지 않으면 2.

유효한 값: 1, 2 또는 6 (6은 입력 내용에 5.1 구성에서 6개의 채널이 있거나 input_audio_channels이 유효한 5.1 구성을 제공하는 경우에만 유효합니다)

호환 가능한 작업 유형: 모두

예: 1

기술:

사용할 오디오 채널 수: 1 (모노) 또는 2 (스테레오).

모노 AAC 오디오는 검사 할 때 때때로 스테레오로 잘못보고됩니다. AAC 오디오의 실제 채널 수를 얻으려면 iTunes를 사용하는 것이 좋습니다. 5.1 채널 오디오 출력도 지원됩니다. 이를 위해output_audio_channels대신 및 을 사용합니다.

  {
    "input": "s3://zencodertesting/test.mov",
    "outputs": [
      {
        "audio_channels": 1
      }
    ]
  }

참조: 오디오_비트레이트 , 오디오_품질 , 입력_오디오_채널및출력_오디오_채널

audio_bit_depth:Integer

부모: outputs

기본: 16

호환 가능한 작업 유형: 모두

예: 24

기술:

각 샘플의 비트 수는 다음과 같습니다. 16, 24, 32.

  {
    "input": "s3://zencodertesting/test.mov",
    "outputs": [
      {
        "audio_bit_depth": 24
      }
    ]
  }

참조: 오디오_샘플_속도

오디오_언어

audio_language:String

API 버전: V1, V2

부모: outputs

유효한 값: ISO 639-1 또는 639-2 언어 코드

호환 가능한 작업 유형: 모두

예: ko

기술:

언어 콘텐츠를 설명하는 출력의 오디오 트랙에 대한 언어 식별자를 설정합니다. ISO 639-1 또는 ISO 639-2 언어 코드 (각각 2 자 또는 3 자) 일 수 있습니다.

참고: 현재 MP4, MKV 및 WEBM 형식 출력과 DASH 스트리밍 전달 출력에 대해 지원됩니다.

  {
    "input": "s3://zencodertesting/test.mov",
    "outputs": [
      {
        "audio_language": "en"
      }
    ]
  }

input_audio_channels

input_audio_channels:Hash

API 버전: V1, V2

부모: outputs

유효한 값: 채널 식별자 및 트랙 인덱스 값의 해시입니다.

호환 가능한 작업 유형: 모두

예:

  • {"FR":1,"FL":2,"FC":3,"BL":4,"BR":5,"LFE":6}
  • {"DL":7,"DR":8}
  • {"FL":"1:1","FR":"1:2"}

기술:

입력의 오디오 트랙과 해당 채널 식별자 간의 매핑을 설정합니다. 이 옵션은 여러 오디오 트랙이있는 입력을 받아 출력에 단일 오디오 트랙을 만드는 데 사용할 수 있습니다. 해시의 키는 유효한 채널 식별자 여야하며 값은 입력의 오디오 트랙 인덱스에 해당하는 정수 여야합니다.

이 옵션은 추가 채널 인덱스를 지정하여 입력에서 개별 채널을 선택하는데도 사용할 수 있습니다. 예를 들어 채널 식별자를 "1:1 “로 설정하면 입력에서 첫 번째 오디오 트랙의 첫 번째 채널이 선택됩니다.

유효한 오디오 채널 식별자: "FL", "FR", "FC", "LFE", "BL", "BR", "DL", "DR".

  {
    "input": "s3://zencodertesting/test.mov",
    "outputs": [
      {
        "input_audio_channels": {
          "FL": 1,
          "FR": 2,
          "FC": 3,
          "LFE": 4,
          "BL": 5,
          "BR": 6
        }
      }
    ]
  }

output_audio_channels

output_audio_channels:Array or Integer

API 버전: V1, V2

부모: outputs

유효한 값: 출력 오디오 채널 수를 정의하는 채널 레이아웃 또는 정수입니다.

호환 가능한 작업 유형: 모두

예:

  • ["플로리다”, "FR”, "FC”, "BL”, "BR”, "LFE "]
  • 2
  • 1
  • 6 (6은 입력 컨텐츠에 5.1 구성에서 6개의 채널이 있거나 input_audio_channels이 유효한 5.1 구성을 제공하는 경우에만 유효합니다.)

기술:

이 옵션을 input_audio_channels옵션과 함께 사용하여 사용자 정의 출력 오디오 채널 레이아웃 또는 믹스다운을 정의합니다.

이 옵션에는 사용자 정의 오디오 채널 레이아웃을 지정하기위한 모드와 모노 또는 스테레오 믹스 다운을 지정하기위한 두 가지 모드가 있습니다.

이 옵션은 유효한 채널 식별자의 배열이거나 정수일 수 있습니다. 스테레오 믹스다운을 만들려면 이 파라미터를 로 설정합니다2 . 모노 믹스다운을 만들려면 이 파라미터를 로 설정합니다1 .

유효한 오디오 채널 식별자: " FL“," FR“," FC“," LFE“," BL“," BR“," DL“," DR”.

이 옵션을 유효한 채널 식별자 배열로 설정하면 사용자 지정 오디오 채널 레이아웃을 만들 수 있습니다.

참고:이 옵션은 audio_channels 보다우선합니다.

  {
    "input": "s3://zencodertesting/test.mov",
    "outputs": [
      {
        "input_audio_channels": {
          "FL": 1,
          "FR": 2,
          "FC": 3,
          "LFE": 4,
          "BL": 5,
          "BR": 6
        },
        "output_audio_channels": [
          "FL",
          "FR",
          "FC",
          "BR",
          "BL",
          "LFE"
        ]
      }
    ]
  }
