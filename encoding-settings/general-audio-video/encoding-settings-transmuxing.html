개요

트랜스 먹싱은 원본 파일의 일부 또는 전체 스트림을 보존하면서 비디오 또는 오디오 파일의 형식을 변경하는 프로세스입니다.

현재 Zencoder는 MP4/MOV 형식에서 MP4/TS로 H.264 비디오 및 AAC/MP3 오디오 트랙을 전송할 수 있습니다 (조각화된 MP4 또는 세그먼트 TS 출력 생성 포함). 그러나 올바르게 트랜스 먹싱하기 위해 입력 파일이 충족해야하는 여러 요구 사항이 있습니다.

비디오

  • MP4/MOV 컨테이너 형식
  • H.264 코덱
  • 사용 된 b- 프레임 없음

오디오

  • MP4/MOV 컨테이너 형식
  • AAC 또는 MP3 코덱

분할 / 조각화

  • 최대 GOP는 세그먼트 기간보다 작거나 같아야합니다.
  • HE-AAC-V2는 사용할 수 없습니다.

다른

  • 스트림이 다시 인코딩되지 않기 때문에 대부분의 인코딩 옵션은 트랜스 먹싱 된 파일에 영향을주지 않습니다.

copy_video

copy_video:Boolean

API 버전: V1, V2

상위: 출력

기본값:거짓

유효한 값:참 또는 거짓

호환 가능한 작업 유형: 모두

예: true

기술:

copy_video지정하면 입력 비디오 파일에서 비디오 트랙을 가져와 결과 출력 파일로 전송합니다.

{
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "copy_video": true
    }
  ]
}
