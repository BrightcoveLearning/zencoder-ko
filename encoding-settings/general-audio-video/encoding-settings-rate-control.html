품질

quality:Integer

API 버전: V1, V2

상위: 출력

기본: 3

유효한 값: 1-5

호환 가능한 작업 유형: 모두

예: 4

기술:

원하는 출력 비디오 품질 (1 ~ 5). 이 품질을 달성하기 위해 video_bitrate를 자동으로 선택합니다.

원하는 품질의 비디오를 생성하는 비디오 비트 전송률을 자동으로 선택합니다. 콘텐츠의 복잡성, 해상도 등으로 인해 하나의 비디오가 목표 화질을 달성하기 위해 다른 비디오보다 높은 비트 전송률을 필요로 할 수 있기 때문에 유용합니다.

quality및video_bitrate가 설정된 경우video_bitrate설정은 무시됩니다.

품질이 높을수록 비트 전송률이 높아지고 그 반대의 경우도 마찬가지입니다.

품질 1: 압축률이 높습니다. 평범한 시각적 품질이지만 작은 파일입니다.
품질 2: 허용되는 품질.
품질 3: 양질. 대부분의 웹 비디오보다 낫습니다.
품질 4: 훌륭한 품질. 훌륭해 보인다.
품질 5: 거의 무손실. 대용량 파일. 이 출력을 다시 인코딩 할 계획이 아니면 권장되지 않습니다.

일반적으로 품질을 한 수준 낮추면 파일 크기가 약 40 % 줄어 듭니다. 고품질 인코딩은 저품질 인코딩보다 약간 느립니다.

화질 설정을 사용할 때 출력 비트 전송률은 비디오 유형에 따라 파일마다 다릅니다. 동일한 품질 설정에서도 스크린 캐스트와 같은 저 복잡성 비디오는 일반적으로 영화와 같은 고 복잡성 비디오보다 비트 전송률이 낮습니다.

  {
    "input": "s3://zencodertesting/test.mov",
    "outputs": [
      {
        "quality": 4
      }
    ]
  }

참조: 비디오_비트레이트

video_bitrate

video_bitrate:Integer

API 버전: V1, V2

상위: 출력

유효한 값: 양의 정수. 최대 100000.

호환 가능한 작업 유형: 모두

예: 1200

기술:

비디오에 대해 원하는 출력 비트 전송률 (kbps로 표시)입니다. 이로 인해 예측 가능한 출력 비트 전송률이 발생하지만 품질은 예측할 수 없습니다. 예를 들어 640x480에서 500kbps이면 동영상 블로그가보기에 충분할 수 있지만 액션 영화는 동일한 비트 전송률에서 나빠 보일 수 있습니다. 마찬가지로 스크린 캐스트에 비해 너무 높기 때문에 필요한 것보다 큰 파일이 생성 될 수 있습니다.

신뢰할 수있는 비트 레이트가 신뢰할 수있는 품질보다 더 중요하다면 video_bitrate를 선택하십시오. 신뢰할 수 있는 시각적 품질이 더 중요하다면품질을대신 사용하십시오.

quality및video_bitrate가 설정된 경우video_bitrate설정은 무시됩니다.

video_bitrate는 2 패스 가변 비트레이트 (VBR) 인코딩을 사용합니다. one_pass 인코딩은 one_pass옵션을 통해 트리거될 수 있지만, one_pass VBR 인코딩은 일반적으로 품질이 좋지 않으며 속도가 작기 때문에 이는 권장되지 않습니다.

또한 너무 높거나 너무 낮은 비트 전송률은 사용할 수 없습니다. 기본적으로 너무 낮은 비트 전송률을 지정하면 허용 가능한 비트 전송률로 자동 수정됩니다. 이 상황에서 자동 수정이 아닌 작업이 실패하도록 하려면엄격모드를 활성화합니다.

  {
    "input": "s3://zencodertesting/test.mov",
    "outputs": [
      {
        "video_bitrate": 1200
      }
    ]
  }

오디오_비트레이트

audio_bitrate:Integer

API 버전: V1, V2

상위: 출력

유효한 값: 양의 정수.

호환 가능한 작업 유형: 모두

예: 96

기술:

Kbps 단위의 출력 비트 전송률 설정입니다. 2 채널 출력의 경우 이는 각 채널의 비트 레이트가 아닌 총 오디오 비트 레이트이므로 128kbps 스테레오 출력은 채널당 64kbps로 인코딩됩니다. 채널당 48kbps 미만에서는 품질이 저하 될 수 있습니다.

audio_quality및audio_bitrate가 설정된 경우audio_bitrate설정은 무시됩니다.

audio_sample_rate를 수동으로 설정하는 경우 호환되는 audio_bitrate를 선택해야 합니다. 그렇지 않으면 인코딩이 실패할 수 있습니다. 기본적으로 이러한 오류를 자동으로 수정하고 유효한 비트 전송률 / 샘플링 속도 조합을 선택합니다. 이 자동 수정을 방지하려면 strict_mode를 활성화하십시오.

이 설정은 인코딩 된 비디오의 최고 비트 전송률을 제한하지 않습니다. 시스템에서 최종 인코딩 된 비트 전송률이이 값보다 높음을 감지하면이 비트 전송률을 대상 평균 비트 전송률로 사용하여 내부적으로 인코딩을 다시 시작합니다. 피크 비트레이트 제한은 을 참조하십시오decoder_bitrate_cap .

AAC 오디오의 경우 84보다 낮은 비트 전송률은 AAC 프로파일이 HE-AAC로 설정됩니다 .

  {
    "input": "s3://zencodertesting/test.mov",
    "outputs": [
      {
        "audio_bitrate": 96
      }
    ]
  }

max_video_bitrate

max_video_bitrate:Integer

API 버전: V1, V2

상위: 출력

유효한 값: 양의 정수. 최대 100000.

호환 가능한 작업 유형: 모두

예: 1500

기술:

영화의 최대 평균 비트 전송률입니다. 품질 및 video_bitrate 설정을 모두 재정 의하여 비트 전송률이 제공된 수를 초과하지 않도록합니다.

max_video_bitrate 는 비트 전송률을 자동으로 선택하는품질설정을 사용하여 모바일 장치를 인코딩할 때 특히 유용합니다. 모바일 장치에는 때때로 고정 된 비트 전송률 제한이 있습니다. 예를 들어 iPhone 3GS의 비트 전송률 제한은 1500kbps입니다.

참고: 이 옵션은 VP6 또는 HEVC 코덱과 호환되지 않습니다.

  {
    "input": "s3://zencodertesting/test.mov",
    "outputs": [
      {
        "max_video_bitrate": 1500
      }
    ]
  }

속도

speed:Integer

API 버전: V1, V2

상위: 출력

기본: 3

유효한 값: 1-5

호환되는 작업 유형: vod

예: 4

기술:

목표 트랜스 코딩 속도입니다. 트랜스 코딩 속도가 느리면 파일 압축이 더 빨라지고 일부 고급 압축 기능을 건너 뛰면 더 빠른 트랜스 코딩이 가능합니다. 유효한 값은 1-5입니다.

참고: 현재 H.264 출력에만 다섯 가지 속도 레벨이 있습니다. VP6 콘텐츠의 경우 1-2는 느린 모드이고 3-5는 빠른 모드입니다. 다른 비디오 코덱은이 설정을 전혀 지원하지 않습니다.

  {
    "input": "s3://zencodertesting/test.mov",
    "outputs": [
      {
        "speed": 4
      }
    ]
  }

디코더 _ 비트 레이트 _ 캡

decoder_bitrate_cap:Integer

API 버전: V1, V2

상위: 출력

유효한 값: 양의 정수. 최대 100000.

호환 가능한 작업 유형: 모두

예: 800

기술:

버퍼를 통해 디코더에 공급되는 최대 비트 전송률입니다. 이 설정은 일반적으로 스트리밍 (RTMP, HLS 또는 브로드 캐스트 비디오)에만 사용됩니다.

비디오 품질을 저하시킬 수 있으므로 의미를 이해하는 경우에만이 설정을 사용하십시오.

라이브 출력의 경우 video_bitrate가 지정된 경우 이 값은 video_bitrate보다 15% 더 높습니다.

  {
    "input": "s3://zencodertesting/test.mov",
    "outputs": [
      {
        "decoder_bitrate_cap": 800
      }
    ]
  }

참고: 디코더_버퍼_크기

디코더 _ 버퍼 _ 크기

decoder_buffer_size:Integer

API 버전: V1, V2

상위: 출력

유효한 값: 양의 정수. 최대 100000.

호환 가능한 작업 유형: 모두

예: 800

기술:

bitrate_cap을 사용할 때 디코더에 공급되는 버퍼의 크기로, kbps로 표현됩니다. buffer_size를 bitrate_cap으로 나눈 값은 버퍼의 크기를 초 단위로 나타냅니다. 따라서 bitrate_cap을 1000으로 설정하고 buffer_size를 1000으로 설정하면 버퍼는 사실상 1.0초입니다. 비트레이트_캡이 500이고 버퍼 크기가 1000이면 버퍼는 2.0초입니다.

비디오 품질을 저하시킬 수 있으므로 의미를 이해하는 경우에만이 설정을 사용하십시오. 일반적으로 스트리밍 (또는 장치 재생)에만 사용해야합니다.

라이브 출력의 경우이 값은 기본적으로 decoder_bitrate_cap의 300 %로 설정되어 3 초의 버퍼 기간을 생성합니다. 그러나 segment_seconds도 설정되면 기본 디코더 _buffer_size가 감소되어 유효 버퍼 기간이 세그먼트 기간의 절반이됩니다.

  {
    "input": "s3://zencodertesting/test.mov",
    "outputs": [
      {
        "decoder_buffer_size": 800
      }
    ]
  }

원_패스

one_pass:Boolean

API 버전: V1, V2

상위: 출력

기본값: false

유효한 값: true또는false

호환되는 작업 유형: vod

예: true

기술:

기본적으로 대상 video_bitrate로 인코딩 할 때마다 2 패스 인코딩을 사용하고 상수품질 인코딩을 수행 할 때 one_pass인코딩을 사용합니다 (두 번째 패스의 이점은 없습니다). 이 옵션은 특정 video_bitrate를 타겟팅 할 때 one_pass 인코딩을 강제합니다.

one_pass 인코딩을 강제로 사용하지 않는 것이 좋습니다. 2 패스 인코딩의 첫 번째 패스는 두 번째 패스보다 빠르므로 2 패스 인코딩에서 one_pass 인코딩으로 이동하면 인코딩이 50% 빨라지지 않고 25% 인코딩 속도 향상이 발생합니다. 그리고 2패스 인코딩은 one_pass 인코딩보다 훨씬 좋습니다.

  {
    "input": "s3://zencodertesting/test.mov",
    "outputs": [
      {
        "one_pass": true
      }
    ]
  }

audio_constant_bitrate

audio_constant_bitrate:Boolean

API 버전: V1, V2

상위: 출력

기본값: false

유효한 값: true또는false

호환 가능한 작업 유형: 모두

예: true

기술:

가능하면 오디오에 대해 일정한 비트레이트 (CBR) 모드를 활성화합니다. 호환성을 위해 audio_sample_rate를 조정할수 있습니다.

이 옵션을 사용할 때 audio_bitrate설정도 지정해야 합니다.

AAC 및 MP3 오디오 코덱과호환됩니다. VP6과 함께 사용할 때는 호환되지 않습니다video_codec .) MP3 오디오의 경우 특정 비트 전송률 만이 옵션과 호환됩니다. 8, 16, 24, 32, 40, 48, 56, 64, 80, 96, 112, 128, 160, 192, 224, 256, 320.

참고: AAC 코덱은 고정 CBR 모드를 정의하지 않으므로 짧은 시간 동안 비트 전송률에 여전히 작은 변화가 있습니다. 대부분의 소프트웨어는 여전히 이러한 인코딩을 VBR로보고합니다.

  {
    "input": "s3://zencodertesting/test.mov",
    "outputs": [
      {
        "audio_constant_bitrate": true
      }
    ]
  }
