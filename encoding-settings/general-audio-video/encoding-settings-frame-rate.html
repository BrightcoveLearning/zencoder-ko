프레임 속도

frame_rate:Float

API 버전: V1, V2

부모: outputs

기본: 원래 (입력) 프레임 속도입니다.

호환 가능한 작업 유형: 모두

예: 25

기술:

10진수로 사용할 출력 프레임 속도 (예: 15 또는 24.98) 입니다.

이 설정은 사용하지 않는 것이 좋습니다. 비디오 프레임 속도를 강제로 변경하면 비디오 품질이 저하됩니다. 특정 수준 이상의 프레임 속도를 방지하려면 다음을 사용하십시오. max_frame_rate대신에.

HD (예: 1920x1080) 보다 큰 해상도에는 60 이상의 프레임 속도가 허용되지 않습니다.

{
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "frame_rate": 25
    }
  ]
}

참고: max_프레임_레이트및데시메이트

max_frame_rate

max_frame_rate:Float

API 버전: V1, V2

부모: outputs

호환 가능한 작업 유형: 모두

예: 30

기술:

출력 프레임 속도를 특정 수로 제한합니다. 그렇지 않으면 프레임 속도가 이 숫자를 초과하면 (예: 입력 프레임 속도가 너무 높음) 프레임 속도가max_frame_rate설정으로 축소됩니다.

예: max_frame_rate 29.97로 설정하고 입력의 프레임 속도가 59.94인 경우 출력 파일의 프레임 속도는 29.97입니다.

다음 두 가지 이유로 정확한 frame_rate를 설정하는 대신 이 설정을 사용하는 것이 좋습니다.

첫째, frame_rate를 변경하면 끊김 현상과 불규칙한 움직임으로 인해 시각적 품질이 저하 될 수 있습니다.

둘째, 프레임 속도를 고정 된 숫자로 설정하면 실제로 출력 비디오의 프레임 속도가 입력의 프레임 속도 이상으로 증가하여 불필요한 파일 크기 증가 (또는 품질 저하)가 발생할 수 있습니다.

{
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "max_frame_rate": 30
    }
  ]
}

예: min_frame_rate 30.0으로 설정하고 입력의 프레임 속도가 29.97인 경우 출력 파일의 프레임 속도는 30.0입니다.

다음 두 가지 이유로 정확한 frame_rate를 설정하는 대신 이 설정을 사용하는 것이 좋습니다.

둘째, 프레임 속도를 고정 된 숫자로 설정하면 실제로 출력 비디오의 프레임 속도가 입력의 프레임 속도 이상으로 증가하여 불필요한 파일 크기 증가 (또는 품질 저하)가 발생할 수 있습니다.

{

데시메이트

decimate:Integer

API 버전: V1, V2

부모: outputs

유효한 값: 양의 정수

호환 가능한 작업 유형: 모두

예: 2

기술:

입력 프레임 속도를 지정된 숫자로 나눕니다. 이것은 프레임 속도를 절반으로 줄이고 싶지만 입력 프레임 속도가 무엇인지 모르는 경우에 유용합니다. 입력 프레임 속도가 20 일 때 decimate를 2로 설정하면 출력 프레임 속도가 10이됩니다. 당신은 여전히 설정할 수 있습니다 max_frame_rate감소된 프레임 속도가 특정 값을 초과하지 않도록 하려는 경우.

출력 비디오에는 입력 프레임 속도를 값으로 나눈 값과 동일한 frame_rate가decimate있습니다. 따라서 입력이 24fps이고 decimate를 4로 설정하면 출력은 6fps (24/4)가됩니다.

{
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "decimate": 2
    }
  ]
}

keyframe_interval

keyframe_interval:Integer

API 버전: V1, V2

부모: outputs

기본: 250

유효한 값: 양의 정수

호환 가능한 작업 유형: 모두

예: 100

기술:

각 키 프레임 사이의 최대 프레임 수를 설정합니다. 기본적으로 키 프레임은 최소 250 프레임마다 생성됩니다. 다른 키 프레임 간격을 지정하면 비디오에서 더 많거나 적은 키 프레임을 만들 수 있습니다. 키 프레임 간격은 양의 정수로 지정해야합니다. 예를 들어 값이 100이면 100 프레임마다 키 프레임이 만들어집니다.

키 프레임 수가 많을수록 출력 파일의 크기가 커지지 만 대부분의 플레이어에서 더 정밀한 스크러빙이 가능합니다.

이것은 최대 키 프레임 간격입니다. 장면 전환 및 기타 사항에 따라 더 많은 키 프레임을 사용하여 고품질 인코딩을 보장 할 수 있습니다.

H.264비디오의 경우 키프레임은 IDR 프레임입니다.

키프레임_인터벌은키프레임_속도보다우선합니다.

{
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "keyframe_interval": 100
    }
  ]
}

참조: 키프레임_레이트

keyframe_rate

keyframe_rate:Float

API 버전: V1, V2

부모: outputs

기본: keyframe_interval 250

호환 가능한 작업 유형: 모두

예: 1.5

기술:

초당 키 프레임 수를 설정합니다. 따라서 값이 0.5이면 2초마다 하나의 키프레임이 생성됩니다. 값이 3이면 초당 세 개의 키프레임이 생성됩니다.

기본값은 를 통해keyframe_interval키프레임을 제어하는 것입니다. 기본값은 250입니다. 비디오 프레임 속도에 따라 8-11 초마다 키 프레임이 생성됩니다.

키 프레임 수가 많을수록 출력 파일의 크기가 커지지 만 대부분의 플레이어에서 더 정밀한 스크러빙이 가능합니다.

keyframe_interval keyframe_rate보다 우선합니다.

{
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "keyframe_rate": 1.5
    }
  ]
}

참조: fixed_keyframe_interval , 강제_키프레임_인터벌 , 강제_키프레임_레이트

fixed_keyframe_interval

fixed_keyframe_interval:Boolean

API 버전: V1, V2

부모: outputs

기본값:거짓

유효한 값:참 또는 거짓

호환 가능한 작업 유형: 모두

예: true

기술:

기본적으로 지정된 은 두 키프레임 사이의 최대 프레임keyframe_interval수만 보장합니다. 인코더는 keyframe_interval에서 요구하기 전에 키 프레임을 배치하기로 결정할 수 있습니다. 예를 들어 장면 변경에 키 프레임을 배치하면 더 나은 압축을 얻을 수 있습니다. 이 옵션을 true로 설정하면 이러한 가변성이 방지됩니다.

이 옵션은H.264 , HEVC또는 VP6을 사용할 때만 적용됩니다. video_codec .

{
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "fixed_keyframe_interval": true
    }
  ]
}

generate_frame_index:Boolean

부모: outputs

기본값:거짓

유효한 값:참 또는 거짓

호환 가능한 작업 유형: 모두

예: true

기술:

프레임 번호, 프레임 크기 (바이트) 및 마지막 키 프레임 번호의 세 가지 정수 값이있는 탭으로 구분 된 파일을 생성합니다. 각 행은 42 열 너비까지 공백으로 채워집니다.

이 옵션은 출력에 비디오 스트림이 포함되어 있지 않으면 사용할 수 없습니다.

{
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "generate_frame_index": true,
      "frame_index_filename": "my_frame_index.txt"
    }
  ]
}
