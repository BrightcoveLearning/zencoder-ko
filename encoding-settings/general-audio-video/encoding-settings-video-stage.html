video_bit_깊이

video_bit_depth:Number

API 버전: V2

상위: 출력

기본: 10

유효한 값: 8, 10 또는 12

호환 가능한 작업 유형: vod, 라이브

예: 5

기술:

색상 채널당 비트 수입니다.

일반적으로 이것을 설정하지 않고 대신 비트 심도를 제공하는 video_codec_profile를 지정합니다.

{
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "video_bit_depth": 10
    }
  ]
}

video_reference_frames

video_reference_frames:String

API 버전: V2

상위: 출력

기본: 3

유효한 값: 0-16, auto

호환 가능한 작업 유형: vod, 라이브

예: 5

기술:

사용할 참조 프레임 수입니다.

참조 프레임이 많을수록 압축 품질이 약간 높아지지만 디코딩 복잡성이 증가합니다. 실제로 5를 초과하는 것은 거의 이점이 없습니다.

에 의해 부분적으로 결정됨속도게다가 video_codec_profile . 속도설정이 자연스럽게 이 숫자를 선택할 수 있도록 하려면 “auto”로 설정하십시오.

압축 및 디코딩 복잡성의 좋은 절충안으로 기본적으로 3을 사용합니다.

레거시 iPod 또는 1세대 iPhone 비디오용으로 제작된 비디오 또는 기술적으로 제한된 기타 디코더에는 1을 사용합니다.

{
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "video_reference_frames": 5
    }
  ]
}

video_codec_profile

video_codec_profile:String

API 버전: V2

상위: 출력

기본: H.264: baseline , 위치: main , VP9: 0

유효한 값:코덱 종속

호환 가능한 작업 유형: 모두

예:높음

기술:

비디오 코덱에서 사용하는 인코딩 프로필을 설정합니다.

  • H.264 프로필: baseline , main , high , high10 , high422 , high444 .
  • HEVC 프로파일: main , main10 , main12 , main422-10main422-12 , main444-8 , main444-10 , main444-12 .
  • VP9 프로필: 0, 1, 2, 3.
  • AV1 프로필: main , high및professional . 기본값: main .

일반적으로 낮은 레벨은 더 넓은 범위의 장치에서 지원되며 디코딩이 더 간단 할 수 있습니다. 레벨이 높을수록 추가 비트 심도 및 크로마 서브 샘플링 선택과 같은 더 많은 기능을 제공합니다. 경우에 따라 더 높은 수준은 압축 효율성을 향상시킵니다.

{
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "video_codec_profile": "high"
    }
  ]
}

video_codec_level

video_codec_level:String

API 버전: V2

상위: 출력

기본: 출력 영상을 기반으로 레벨 차트를 사용하여 자동 계산됩니다. H.264 레벨 , HEVC 레벨 , AV1 레벨을 참조하십시오 .

유효한 값: 유효한 수준: H.264 레벨 , HEVC 레벨 , AV1 레벨 .

호환 가능한 작업 유형: 모두

예: 4.1

기술:

비트 레이트, 매크로 블록 (H.264) 또는 비트 레이트, 코딩 트리 단위 (HEVC)를 제한하고, 비트 레이트 및 코딩 트리 단위 (CTU) (AV1)를 제한합니다.

주로 기기 호환성에 사용됩니다. 예를 들어, 아이폰은 H.264 레벨 3을 지원합니다. 즉, 비디오의디코더_비트레이트 캡은 10,000kbps를초과할 수 없습니다. 일반적으로이 설정이 필요한 특정 장치를 대상으로하는 경우에만이 설정을 변경해야합니다.

{
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "video_codec_level": "4.1"
    }
  ]
}

video_bframes

video_bframes:Integer

API 버전: V2

상위: 출력

기본: 0

유효한 값: 0-16

호환 가능한 작업 유형: 모두

예: 2

기술:

연속 B 프레임의 최대 수.

B- 프레임을 사용하면 동일한 비트 전송률에서 품질을 높일 수 있지만 CPU에 더 많은 부담이 있습니다. 이것은 오래된 컴퓨터 나 장치에서는 문제가 될 수 있지만 일반적으로 최신 하드웨어에서는 원활하게 재생됩니다.

H.264는 베이스라인이 아닌기본 또는 하이 프로파일이있는 b 프레임으로 지원합니다. 가장 광범위한 호환성을 위해 기본값은 0입니다.

HEVC의 경우 b- 프레임은 모든 수준에서 사용할 수 있습니다. 기본값은 3입니다.

압축 / 품질 향상을 위해 값 3을 권장합니다. 5 또는 6보다 큰 값은 거의 이점을 제공하지 않으며 인코딩 시간이 크게 늘어납니다.

{
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "video_bframes": 2
    }
  ]
}

참조: 비디오_코덱_프로필

color_metadata

color_metadata:String

API 버전: V2

상위: 출력

기본값: preserve

유효한 값: preserve또는discard

호환 가능한 작업 유형: 모두

예: discard

기술:

이 옵션은 현재 H.264, HEVC 및 VP9 출력에서 지원됩니다. 입력 비디오에 행렬 계수, 전송 특성 및 색상 기본 메타데이터가 있는 경우 Zencoder는 기본적으로preserve이러한 설정을 사용합니다.

이 메타데이터가 출력에 포함되지 않도록color_metadata하려면 로 설정합니다discard .

{
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "color_metadata": "discard"
    }
  ]
}
