개요

Zencoder는 다양한 형태의 적응 형 및 세그먼트 스트리밍을 지원합니다.

  • HTTP 라이브 (분할) 스트리밍 및 재생 목록
  • Microsoft Smooth Streaming
  • 고풍 SMIL

각 스트리밍 형식은 아래 나열된 API 옵션의 다른 하위 집합을 사용합니다. 자세한 내용은 각 스트리밍 유형에 대한 가이드를 참조하세요.

참고: 일부 플레이어는 오디오 및 비디오 스트림 길이가 세그먼트 길이보다 많이 다른 경우를 처리하지 않습니다. 이 문제가 발생하면 더 짧은 세그먼트 길이를 사용해 보십시오.

메모

  • 작업에는 하나 이상의 표준 또는 세그먼트 출력이 있어야합니다. 재생 목록 출력만으로 작업을 생성하는 것은 작동하지 않습니다.
  • 재생 목록 출력은 url ,, 를 제외한 대부분의 출력 옵션을 무시합니다. base_urlfilename , streams , public , 및type .
  • 세그먼트화된 출력이 임시 저장소에 업로드되면 ZIP 아카이브로 업로드됩니다 .

스트리밍_배달_형식

streaming_delivery_format:String

API 버전: V1, V2

부모: outputs

유효한 값:대시

호환 가능한 작업 유형: 모두

예:대시

기술:

필요한 매니페스트, 디렉터리 구조 또는 기타 패키징을 포함하여 특정 스트리밍 구성을 사용하여 전달 될 출력의 형식 / 프로토콜을 설정합니다.

'대시' 출력의 경우 기본 segment_seconds 를 2로 변경하고, 고정 키프레임 간격을 적용하고, MPD 매니페스트를 포함하며, 초기화 및 세그먼트 파일을 사용하여 디렉토리 구조를 만듭니다.

{
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "type": "segmented",
      "streaming_delivery_format": "dash"
    }
  ]
}

참조: 스트리밍_배달_프로필

스트리밍_배달_프로필

streaming_delivery_profile:String

API 버전: V1, V2

부모: outputs

기본값:라이브

유효한 값:라이브, hbbtv_1.5, 온_수요

호환 가능한 작업 유형: 모두

예:라이브

기술:

스트리밍 전달 형식의 프로필을 설정하여 프로필과의 호환성을위한 옵션을 선택합니다. 유효한 옵션은 선택한 옵션에 따라 다릅니다streaming_delivery_format .

dash출력의 경우 현재 지원되는 프로파일은 다음과 같습니다.

  • live - DASH프로파일을 “urn:mpeg:대시:프로파일:isoff-live:2011"로 설정하고SegmentTemplate각각에 대해 단일 프로파일을 사용합니다. AdaptationSet . DRM의 경우 8바이트 IV를 사용하고 에서는 미디어의 PSSH 원자를 포함하여 버전 1.1을 사용합니다. PlayReady
  • hbbtv_1.5 - DASH프로파일을 “urn:hbbtv:dash:프로파일:isoff-live:2012"로 설정하고 오디오 및 비디오 표현에 필요한 속성을 포함합니다. DRM의 경우 16 바이트 IV를 사용합니다. PlayReady PSSH 원자를 제외한 버전 1.0을 사용합니다.
  • on_demand - DASH프로파일을 “urn:mpeg: dash:profile:isoffon-demand:2011"로 설정하고 미디어를 세그먼트화하는 대신 각 스트림에 대해 단편화된 단일 파일로 인코딩합니다. 그렇지 않으면 라이브 프로필과 동일합니다.
{
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "type": "segmented",
      "streaming_delivery_format": "dash",
      "streaming_delivery_profile": "hbbtv_1.5"
    }
  ]
}

참조: streaming_delivery_format

segment_seconds

segment_seconds:Float

API 버전: V1, V2

부모: outputs

기본: 10

유효한 값: 1.0에서 3600.0 사이의 숫자로, 2.0보다 작은 값은byte_range_segmenting가 활성화된 경우에만 허용됩니다. 매우 짧은 세그먼트가 재생 문제를 일으키는 것으로 확인되었으므로 최종 세그먼트가 지정된 값보다 길 수 있습니다.

{

  "outputs": [

      "type": "segmented",
      "segment_seconds": 8
    }
  ]

대체_오디오

alternate_audio:Hash

API 버전: V1, V2

부모: outputs

유효한 값: 대체 오디오 그룹의 해시 (각각 오디오 스트림 변환 해시의 배열)

호환 가능한 작업 유형: 모두

기술:

HLS재생 목록에 대한 대체 오디오 스트림 세트를 제공합니다. 4 이상이어야max_hls_protocol_version합니다.

이 옵션은 재생 목록에EXT-X-MEDIA태그를 삽입합니다. 를name설정하지 않으면 주어진 이름을 기반으로 이름이 생성됩니다language .

대체 오디오 변환에 유효한 옵션

  • language: "en", "es"또는 "zh"와 같은 오디오 트랙의 언어에 대한 ISO 639 코드입니다.
  • name (선택 과목): 대체 오디오 그룹의 이름입니다. 이름이 설정되지 않은 경우 기본적으로 언어 이름이 사용됩니다.
  • path (선택 과목): 이 오디오 변환을위한 M3U8 재생 목록의 선택적 경로입니다. 경로가 설정되지 않은 경우이 대체 오디오 스트림은 소스 스트림에 포함 된 오디오를 설명합니다.
  • source (선택 과목): 동일한 작업의 미디어 출력에있는 레이블입니다. 이 오디오 스트림을 사용하는 렌 디션의 코덱을 설정하는 데 사용할 수 있습니다.
{
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "type": "playlist",
      "alternate_audio": {
        "custom_audio_group": [
          {
            "source": "english-audio",
            "path": "alternate-audio-en.m3u8",
            "language": "en"
          },
          {
            "source": "french-audio",
            "path": "alternate-audio-fr.m3u8",
            "language": "fr"
          }
        ],
        "secondary_audio_group": {
          "path": "path/to/secondary-audio-group.m3u8",
          "language": "en",
          "name": "Secondary English Audio Track",
          "source": "secondary-english-audio"
        }
      }
    }
  ]
}

참고: HLS버전 6 및 7은 완전히 지원되지 않습니다. 유일한 효과에 대해 버전 6과hls_protocol_version 7을 지정하는 경우 제거해야 합니다.재생 목록에서 더 이상 사용되지 않는 기능.

호환 가능한 작업 유형: 모두

예: 3

기술:

사용할 HLS 프로토콜 버전을 설정합니다. 사용 가능한 특수 기능이 사용됩니다.

참고: HLS 버전 1-3은 정수를 사용합니다. duration HLS 재생 목록에서. 버전 4 이상에서는 부동 소수점 숫자를 사용합니다.

참고: HTTP 라이브 스트리밍출력에만 적용됩니다.

{
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "type": "segmented",
      "hls_protocol_version": 3
    }
  ]
}

참조: max_hls_protocol_version

생성_키프레임_매니페스트

generate_keyframe_manifest:Boolean

API 버전: V2

부모: outputs

기본값: false

유효한 값: true또는false

호환 가능한 작업 유형: 모두

예: true

기술:

빨리 감기 및 역방향 재생에 필요한HLS키프레임 (i-frame) 매니페스트를 생성합니다.

기본 키프레임 매니페스트 파일 이름은 재생 목록 파일 이름과 동일하지만 '-iframe-index 가' 추가되었습니다 (따라서 'master.m3u8' 재생 목록의 경우 키프레임 매니페스트는 'master-iframe-index.m3u8'가 됩니다). 키프레임 매니페스트 파일 이름은 로 재정의할 수keyframe_manifest_filename있습니다.

max_hls_protocol_version또는 경우 기본적으로 활성화됩니다. hls_protocol_version이 (가) 4 이상으로 설정되어 있습니다.

{
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "type": "segmented",
      "format": "ts",
      "generate_keyframe_manifest": true,
      "keyframe_manifest_filename": "my_iframe_index.m3u8"
    }
  ]
}

참조: keyframe_manifest_filename

keyframe_manifest_filename

keyframe_manifest_filename:String

API 버전: V2

부모: outputs

기본값: iframe_index.m3u8

호환 가능한 작업 유형: 모두

예: my_iframe_index.m3u8

기술:

지정하지 않는 한 키프레임 매니페스트에 사용되는 파일 이름은 재생 목록 파일 이름과 동일하지만 '-iframe-index가' 추가됩니다. 예를 들어 재생 목록 'master.m3u8'의 경우 키 프레임 매니페스트는 'master-iframe-index.m3u8'이됩니다.

이 옵션을 사용하려면 generate_keyframe_매니페스트가 참이거나hls_protocol_version 4 이상이어야 합니다.

{
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "type": "segmented",
      "format": "ts",
      "generate_keyframe_manifest": true,
      "keyframe_manifest_filename": "my_iframe_index.m3u8"
    }
  ]
}
