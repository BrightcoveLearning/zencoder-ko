type:String

API 버전: V1, V2

상위: 출력

기본값:표준

유효한 값: standard , segmented , captions , playlist , 및transfer-only

호환 가능한 작업 유형: 모두

예: segmented

기술:

기본적으로 Zencoder는 출력을 표준 오디오 또는 비디오 파일로 간주합니다. 현재 두 가지 특수 유형의 출력값이segmented있습니다playlist .

  • segmented: 세그멘테이션의 일반적인 사용 사례는 Apple의 라이브 HTTP 스트리밍을 위한 것입니다 ( 세그먼트 스트리밍 참조 ). 세그먼트 출력의 경우 기본 형식이 mp4 대신 ts로 변경됩니다.
  • captions: API V2 전용
    캡션 출력 유형은 캡션이 추가된 비디오가 아니라 소스 비디오의 캡션만 포함하는 파일을 생성합니다. 참고: 캡션 유형 출력만 작업에 대한 출력은 아닐 수 있습니다.
    캡션은 별도의 파일 ( caption_url옵션 사용) 으로 Zencoder에 제공되거나 기본적으로 소스 비디오에서 추출됩니다.
    소스 비디오에서 캡션을 추출하기 위해 지원되는 형식은 MP4 및 TS입니다. MP4 파일에서 캡션은 별도의 트랙 (CEA-608 형식)에있을 수 있습니다. 그렇지 않으면 MP4 또는 TS 파일의 캡션은 A / 53 또는 SCTE-128 (CC1 및 CC3) 표준을 준수하는 MPEG2 또는 AVC 비디오 스트림에 포함되어야합니다.
    캡션 유형 출력에 지원되는 출력 형식: ' mp4 ', ' scc ', ' webvtt ',' vtt ', m3u8 ', ' dfxp '.
  • 재생 목록 : 재생 목록 출력 유형은 마스터 M3U8 재생 목록 파일을 생성하기 위해 다중 비트 전송률 출력 스트림에 사용됩니다. 참고: 재생 목록 유형 출력이 작업에 대한 유일한 출력은 아닐 수 있습니다.
  • transfer-only: API V2 전용
    transfer-only출력은 인코딩을 건너뛰고 원본 비디오를 URL로 보냅니다 .
    transfer-only출력은소스를사용하여 다음을 사용하는 대신 다른 출력에서 지정된 URL로 비디오를 전송할 수도 있습니다.입력.
{
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "type": "segmented"
    }
  ]
}

url:String

API 버전: V1, V2

상위: 출력

유효한 값: 유효한 S3, Cloud Files, GCS, FTP, FTPS, SFTP, Aspera, Azure 또는 HTTP 주소.

호환 가능한 작업 유형: 모두

예:

  • S3: //my-output-bucket/final.mp4
  • cf: //사용자명:api_key @container /객체
  • cf+영국: //사용자 이름:api_key @container /개체
  • gcs: //액세스_키:비밀_키 @gcs -버킷/객체
  • ftp://user:password@ftp.example.com/final.mp4
  • sftp://user:password@sftp.example.com/final.mp4
  • 아스페라: //사용자:암호 @호스트:ssh-포트/경로/에/파일.mp4

기술:

출력 파일이 업로드 될 파일 이름을 포함한 전체 URL입니다. 봐주세요 base_url제공업체에 대한 자세한 내용은

S3, GCS, Cloud Files 및 Akamai에 대한 HTTP 업로드를 공식적으로 지원합니다.

S3에 대한 참고 사항:

당신은해야합니다승인하다쓰다출력 버킷에 대한 Zencoder 권한 Zencoder에 작업을 제출하기 전에 쓰기 권한이 필요한 S3 "경로"가 아니라 S3 버킷입니다. 예를 들어, http://s3.amazonaws.com/my_files/0000/video.mp4필요한 자산은 “my_files/0000" (경로) 가 아닌 “my_files" (버킷 이름) 에 쓰기 액세스가 추가됩니다.

URL (? 또는 #)에서 유효하지 않은 특수 문자를 포함하는 키를 만들려면 URL 인코딩 (각각 % 3F 및 % 23)해야합니다. 다른 모든 문자는 URL 인코딩되지 않아야합니다.

URL의 프로토콜을 로s3+region-name설정하여 S3 버킷의 리전을 제공할 수 있습니다. 여기서 “region-name”은이 문서에서 정의됩니다 . 예를 들어 아시아 태평양 (싱가포르) 리전의 버킷에서 파일에 액세스하려면 URL을 로 설정합니다s3+ap-southeast-1://my-bucket/file.mp4 .

클라우드 파일에 대한 참고 사항:

같은 프로토콜에 추가하여 사용할 리전 (DFW, ORD 또는 UK) 을 지정할 수cf+ord://username:api_key@container/object있습니다. 현재 지원되는 유일한 Cloud Files 지역은 DFW, ORD 및 UK입니다. 기본적으로 DFW 영역이 사용됩니다.

컨테이너 이름의 모든 특수 문자는 URL로 인코딩되어야합니다.

Google 클라우드 스토리지에 대한 참고 사항:

GCS 액세스 키는 Google API 콘솔의 “상호 운용 가능한 액세스” 섹션에서 찾을 수있습니다. 액세스 키는제대로 이스케이프해야합니다.

점이 포함 된 GCS 버킷 이름은 지원되지 않습니다.

Azure 참고 사항:

URL에 특수 문자를 사용한 인증이 포함된 경우그들이 제대로 탈출했는지 확인하십시오 .

FTP에 대한 참고 사항

URL에 특수 문자를 사용한 인증이 포함된 경우그들이 제대로 탈출했는지 확인하십시오 .

서버의 루트에서 쓰기를 시도하므로 URL에 대한 절대 경로를 사용하여 서버에 성공적으로 쓸 수 있는지 확인하십시오.

아스페라에 대한 참고 사항:

URL의 ssh-port 부분은 필요하지 않으며 기본값은 22입니다.

Zencoder는 비디오 인코딩 서버에서 Aspera 클라이언트를 실행합니다. 이 서버는 파일을 호스팅하는 Aspera 엔터프라이즈 또는 연결 서버에연결됩니다.

이는 파일을 사용할 수 있도록 FTP 서버를 실행하고 파일 전송을 위해 FTP 클라이언트를 사용하는 Zencoder와 유사한 상황입니다.

HTTPS를 통한 Akamai 넷스토리지에 대한 참고 사항:

Zencoder는 FTP 또는 HTTPS를 통해 Akamai NetStorage에 콘텐츠를 업로드 할 수 있습니다. HTTPS를 통해 콘텐츠를 업로드하려면 다음과 같은 출력 URL을https://[Host]-nsu.akamaihd.net/[CPCode]/[path]/[file.ext]생성하십시오.

Akamai NetStorage 키 이름 및 키를 다음과 같이 참조할 수 있습니다. 외부 서버신임장설정하여외부 서버 사용자필드를 Akamai NetStorage 키 이름 및외부 서버 비밀번호 Akamai NetStorage 키로.

{
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "url": "s3://my-output-bucket/final.mp4"
    }
  ]
}

strict:Boolean

API 버전: V1, V2

상위: 출력

기본값: false

유효한 값: true또는false

호환 가능한 작업 유형: 모두

예: true

기술:

" strict " 모드를 활성화합니다. 기본적으로 인코딩 문제를 해결하려고합니다. 이 모드를 " compatible " 모드라고 합니다. 예를 들어 인코더에 대해 너무 낮은 비디오 비트 전송률을 설정하면 가능한 가장 낮은 비트 전송률을 자동으로 수정하고 사용합니다. 따라서 150kbps의 비트 전송률을 요청했지만 인코더가 200kbps 미만의 작업을 수행 할 수없는 경우 자동으로 비디오를 200kbps로 인코딩합니다.

“strict " 모드를 활성화하면 이와 같은 인코딩 설정을 수정하지 않습니다. 대신 대상 사양을 충족 할 수 없기 때문에 작업이 실패합니다.

{
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "strict": true
    }
  ]
}

{

  "output": [
    {
      "label": "parent",
      "url": "s3://bucket/video.mp4"
    },
    {
      "source": "parent",
      "url": "s3://other-bucket/video.mp4",
      "type": "transfer-only"
    }
  ]
}

{
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "url": "s3://bucket/video.mp4",
      "credentials": "s3_production"
    }
  ]
}

parallel_upload_limit

parallel_upload_limit:Integer

API 버전: V1, V2

기본: S3의 경우 30, 기타 모든 대상의 경우 10.

유효한 값: 1에서 30 사이의 정수

호환되는 작업 유형: vod

예: 5

기술:

멀티 파트 출력, 주로 HLS 세그먼트를 업로드 할 때 수행되는 최대 동시 업로드 수입니다.

parallel_upload_limit전체 작업이 아닌 개별 출력에 대해 설정해야 합니다.

 

원격 서버의 대역폭에 따라 전송 시간이 빨라질 수 있습니다. 더 많은 연결이 서버에 더 많은 부하를 줄 수 있습니다. 업로드 시간 초과에 문제가 있거나 Zencoder가 파일을 업로드 할 때 너무 많은 대역폭을 사용하지 않도록하려면이 값을 1로 설정하십시오.

{
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "parallel_upload_limit": 5,
      "url": "s3://my-output-bucket/final.mp4",
      "credentials": "s3_production"
    },
    {
      "parallel_upload_limit": 5,
      "device_profile": "mobile/advanced",
      "url": "s3://my-output-bucket/mobile.mp4",
      "credentials": "s3_production"
    }
  ]
}

헤더

headers:Hash

API 버전: V1, V2

상위: 출력 OR 축소판

호환 가능한 작업 유형: 모두

예: { "Content-Type": "binary/octet-stream" }

기술:

파일을 업로드 할 때 파일과 함께 보낼 HTTP 헤더입니다. 이 기능은 현재 S3, GCS 및 Cloud Files를 사용할 때 지원됩니다.

Zencoder는 다음 헤더의 제한된 하위 집합을 설정할 수 있도록 지원합니다. Cache-Control , Content-Disposition , Content-Encoding , Content-TypeExpires , x-amz-acl , x-amz-storage-class , x-amz-server-side-encryption , x-amz-meta-* , x-goog-acl 및x-goog-meta-* .

메모:
  • S3의 Standard_IA 기능을사용하려면x-amz-storage-class헤더를 적절한 값으로 설정해야 합니다. 이러한 값은 변경되고 AWS 및 Google에서 제어하므로 올바른 값을 가져와야합니다.
  • "goog"헤더는 GCS 출력 URL에 대해 지원되지만 S3의 URL에는 지원되지 않습니다.

출력 위치가 지정되지 않으면 헤더가 무시됩니다.

{
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "url": "s3://my-bucket/file.mp4",
      "headers": {
        "Content-Type": "binary/octet-stream",
        "x-amz-acl": "public-read-write"
      }
    }
  ]
}
