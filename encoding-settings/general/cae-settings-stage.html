개요

컨텍스트 인식 인코딩은 사용자가 제공 한 지침을 기반으로하지만 비디오 자체의 특성을 기반으로하여 소스 비디오의 여러 변환을 만듭니다. 이렇게하면 소스의 특성을 고려할 때 본질적으로 가치가없는 변환을 만들어 비디오를 트랜스 코딩하고 저장하는 비용을 피할 수 있습니다.

참조 CAE 가이드설정에 대한 추가 정보 및 권장 사항은

메모:
  • 모든 Zencoder 고객은 컨텍스트 인식 인코딩을 사용할 수 없습니다. 컨텍스트 인식 인코딩을 사용할 수 있는지 여부에 대한 질문이 있으면 계정 관리자에게 문의하십시오.

video_codecs

video_codecs:Array

API 버전: V2

부모: dynamic_profile_options

유효한 배열 항목 값: h264 , hevc

기본값:없음

호환 가능한 작업 유형: VOD

예: ["h264", "hevc"]

기술:

혼합 코덱 출력의 경우 변환을 만들 코덱의 배열입니다. 현재 허용되는 항목 값은h264및 입니다hevc .

{
    "dynamic_profile_options": {
      "max_renditions": 6,
      "video_codecs" : ["h264", "hevc"'],
      "video_codec_options": {
        "h264": {
          "min_renditions": 2,
          "max_resolution": { "width": 1280, "height": 720 }
        },
        "hevc": {
          "min_renditions": 2,
          "max_resolution": { "width": 1920, "height": 1080 }
        }
      }
    }
  }

video_codec_options

video_codecs:Object

API 버전: V2

부모: dynamic_profile_options

기본값:없음

호환 가능한 작업 유형: VOD

예:

"video_codec_options": {
  "h264": {
    "min_renditions": 2,
    "max_resolution": { "width": 1280, "height": 720 }
  },
  "hevc": {
    "min_renditions": 2,
    "max_resolution": { "width": 1920, "height": 1080 }
  }
}

기술:

혼합 코덱 출력의 경우 각 코덱에 대한 옵션을 지정하는 개체입니다. 각 속성은 이름이 있는h264객체이거나hevc , 해당 속성은 해당 코덱에 대한 변환에 대한 제약 조건을 설정합니다. 이러한 개체에 허용되는 속성은 다음과 같습니다.

  • min_renditions필수
  • min_bitrate
  • max_bitrate
  • max_first_rendition_bitrate
  • min_resolution
  • max_resolution
  • max_first_rendition_resolution
참고: 이 설정은 전역 수준에서도 사용됩니다. 전역 및 코덱 별 수준 모두에서 설정을 사용하는 경우 값이 호환되어야합니다. 예를 들어 전역 수준에서 설정된 max_renditions은 코덱 수준에서 설정된 min_rendition 값의 합계보다 크거나 같아야 합니다.
{
    "dynamic_profile_options": {
      "max_renditions": 6,
      "video_codecs" : ["h264", "hevc"'],
      "video_codec_options": {
        "h264": {
          "min_renditions": 2,
          "max_resolution": { "width": 1280, "height": 720 }
        },
        "hevc": {
          "min_renditions": 2,
          "max_resolution": { "width": 1920, "height": 1080 }
        }
      }
    }
  }

quality_rate_tune

quality_rate_tune:Number

API 버전: V2

부모: dynamic_profile_options

유효한 값: -40.0 - 40.0

기본값: 0

호환 가능한 작업 유형: VOD

예: 2

기술:

1에서 40까지의 양수로 설정하면quality_rate_tune CAE 프로파일의 비디오 품질이 높아지고 비트 전송률이 높아집니다. 마찬가지로, -1에서 -40 범위의 숫자가 주어지면 비디오 품질이 저하되는 대신 비트 전송률을 낮출 수 있습니다.

예를 들어 -10을 사용하면 약 10 % 낮은 비트 전송률을 기대할 수 있습니다. 10을 사용하면 일반적으로 비트 전송률이 약 10 % 증가합니다. 양수 값에 대한 CAE는 매우 높은 비디오 품질을 보장하는quality_rate_tune데 추가 예방 조치를 취하고 세분성 및 수와 같은 비트 전송률을 제한하는 다른 매개 변수가있을 수 있으므로 이러한 숫자는 근사치입니다.변환.

일반적인 콘텐츠 및 원하는 비디오 품질과 대역폭이 크게 다르기 때문에 다른 값으로 실험하는 것이 좋습니다. 

        {
          "input": "s3://zencodertesting/test.mov",
          "generate_dynamic_profile": true,
          "dynamic_profile_options": {
            "min_renditions": 2,
            "max_renditions": 10,
            "quality_rate_tune": 2
          }
        }

비디오 구성 설정

CAE가 출력에 포함해야하는 특정 구성을 생성하지 않는 경우이 기능을 사용할 수 있습니다.

CAE가 출력의 일부로 생성해야 하는 비디오 해상도 및 코덱 설정을 설명하는 객체 배열입니다 (위예제 참조 ). 이 배열은선택 사항입니다 . 생략하면 192x108부터 콘텐츠 해상도까지의 기본 해상도 배열이 CAE에 대한 기본 입력 세트로 사용됩니다.
