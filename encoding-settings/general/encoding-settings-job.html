API_키

api_key:String

API 버전: V1, V2

유효한 값: https://app.zencoder.com/api 에서 제공하는 젠코더 API 키

호환 가능한 작업 유형: 모두

예: bcafecadfe 카바 페닥

기술:

Zencoder 계정의 API 키입니다. API 키는 https://app.zencoder.com/api 에서 찾을 수있습니다. 해당 페이지에서 API 키를 다시 생성 할 수도 있습니다.

  {
    "input": "s3://zencodertesting/test.mov",
    "api_key": "bcafecadfecabafedac"
  }

dynamic_profile_options

dynamic_profile_options:Object

API 버전: V2

호환 가능한 작업 유형: 모두

기술:

CAE를 사용하면 이러한 출력의 출력 수와 매개 변수가 각 입력 파일에 대해 동적으로 변경 될 수 있습니다. dynamic_profile_options설정은 변환을 만드는 방법을 정의합니다. 자세한 내용은 CAE 설정을참조하십시오. 또한 이 기능을 사용하려면 generate_dynamic_profile와true같게 설정해야 합니다.

  {
    "input": "s3://zencodertesting/test.mov",
    "generate_dynamic_profile": true,
    "dynamic_profile_options": {
      "min_renditions": 2,
      "max_renditions": 8
    },
    "outputs" :[
      {"dynamic_profile_rendition": 1},
      {"dynamic_profile_rendition": 2},
      {"dynamic_profile_rendition": 3},
      {"dynamic_profile_rendition": 4},
      {"dynamic_profile_rendition": 5},
      {"dynamic_profile_rendition": 6},
      {"dynamic_profile_rendition": 7},
      {"dynamic_profile_rendition": 8}
  ]
  }

생성_동적_프로필

generate_dynamic_profile:Boolean

API 버전: V2

유효한 값:참 또는 거짓

기본값: false

호환 가능한 작업 유형: 모두

기술:

CAE를 사용하면 이러한 출력의 출력 수와 매개 변수가 각 입력 파일에 대해 동적으로 변경 될 수 있습니다. 생성할 출력 수와 다른 프로파일 제약 조건에 대한 제한을 지정하려면 CAE 작업 요청에 dynamic_profile_options객체도 포함해야 합니다. CAE 사용을 위해 예약된 출력에는resolutionbitrate , 또는 코덱 관련 매개 변수가 포함되지 않아야 합니다. 대신 라벨을 포함해야dynamic_profile_rendition합니다. CAE 사용을 위해 예약된 출력 수는 다음과 같이max_renditions지정된 수와 일치해야 합니다. dynamic_profile_options .

  {
    "input": "s3://zencodertesting/test.mov",
    "generate_dynamic_profile": true,
    "dynamic_profile_options": {
      "min_renditions": 2,
      "max_renditions": 8
    },
    "outputs" :[
       {"dynamic_profile_rendition": 1},
       {"dynamic_profile_rendition": 2},
       {"dynamic_profile_rendition": 3},
       {"dynamic_profile_rendition": 4},
       {"dynamic_profile_rendition": 5},
       {"dynamic_profile_rendition": 6},
       {"dynamic_profile_rendition": 7},
       {"dynamic_profile_rendition": 8}
    ]
  }

입력

input:String

API 버전: V1, V2

유효한 값: 인증이 있거나없는 미디어 파일 (HTTP / HTTPS, FTP / FTPS, SFTP, Azure, GCS, CF 또는 S3)에 대한 유효한 URL

호환되는 작업 유형: vod

예:

  • http://example.com/path/to/input.avi
  • s ftp://user:pass@example.com/path/to/input.mp3
  • S3: //my-bucket/video.mp4
  • cf: //사용자명:API_키 @container /video.wmv
  • CF+영국: //사용자 이름:API_키 @container /video.wmv
  • 푸른: //계정-이름:계정 키 @container /video.wmv
  • gcs: //액세스_키:비밀_키 @gcs -bucket/video.mov
  • 아스페라: //사용자:패스 @호스트:ssh-포트/경로/에/파일.mp4
  • s3+EU-센트럴-1: //my-bucket/video.mp4

기술:

트랜스 코딩 할 파일의 URL입니다. 원격 서버에 인증이 필요한 경우 입력 URL 문자열에 사용자 이름과 비밀번호를 포함해야합니다.

URL에 특수 문자를 사용한 인증이 포함된 경우그들이 제대로 탈출했는지 확인하십시오 .

S3에 대한 참고 사항:

당신은해야합니다승인하다쓰다출력 버킷에 대한 Zencoder 권한 Zencoder에 작업을 제출하기 전에 쓰기 권한이 필요한 S3 "경로"가 아니라 S3 버킷입니다. 예를 들어, http://s3.amazonaws.com/my_files/0000/video.mp4필요한 자산은 “my_files/0000" (경로) 가 아닌 “my_files" (버킷 이름) 에 쓰기 액세스가 추가됩니다.

URL (? 또는 #)에서 유효하지 않은 특수 문자를 포함하는 키를 만들려면 URL 인코딩 (각각 % 3F 및 % 23)해야합니다. 다른 모든 문자는 URL 인코딩되지 않아야합니다.

S3 URL이 사전 서명된 경우 (“서명”, “만료” 및 “awsAccessKeyID” 쿼리 매개 변수 포함) 모든 문자는 URL로 인코딩되어야 합니다.

URL의 프로토콜을 로s3+region-name설정하여 S3 버킷의 리전을 제공할 수 있습니다. 여기서 “region-name”은이 문서에서 정의됩니다 . 예를 들어 아시아 태평양 (싱가포르) 리전의 버킷에서 파일에 액세스하려면 URL을 로 설정합니다s3+ap-southeast-1://my-bucket/file.mp4 .

Azure 참고 사항:

URL에 특수 문자를 사용한 인증이 포함된 경우그들이 제대로 탈출했는지 확인하십시오 .

클라우드 파일에 대한 참고 사항:

같은 프로토콜에 추가하여 사용할 리전 (DFW, ORD 또는 UK) 을 지정할 수cf+ord://username:api_key@container/object있습니다. 현재 지원되는 유일한 Cloud Files 지역은 DFW, ORD 및 UK입니다. 기본적으로 DFW 영역이 사용됩니다.

컨테이너 이름의 모든 특수 문자는 URL로 인코딩되어야합니다.

Google 클라우드 스토리지에 대한 참고 사항:

GCS 액세스 키는 Google API 콘솔의 “상호 운용 가능한 액세스” 섹션에서 찾을 수있습니다. 액세스 키는제대로 이스케이프해야합니다.

점이 포함 된 GCS 버킷 이름은 지원되지 않습니다.

아스페라에 대한 참고 사항:

URL의 ssh-port 부분은 필요하지 않으며 기본값은 22입니다.

Zencoder는 비디오 인코딩 서버에서 Aspera 클라이언트를 실행합니다. 이 서버는 파일을 호스팅하는 Aspera 엔터프라이즈 또는 연결 서버에연결됩니다.

이는 파일을 사용할 수 있도록 FTP 서버를 실행하고 파일 전송을 위해 FTP 클라이언트를 사용하는 Zencoder와 유사한 상황입니다.

  {
    "input": "http://example.com/path/to/input.avi"
  }

참조: url

출력

outputs:Array or Hash

API 버전: V1, V2

유효한 값: 해시 배열 또는 단일 해시

호환 가능한 작업 유형: 모두

예:

  • “출력”: [ {} ]
  • "산출": {}

기술:

기본 출력은 다음 설정을 가진 단일 비디오 파일입니다.

  • H.264 비디오, AAC 오디오, MP4 파일 형식
  • 비디오: 중간 품질
  • 오디오: 스테레오, 중간 품질 (약 112kbps), 44100Hz

출력 위치를 지정하지 않으면 파일이 Zencoder에서 임시로 호스팅됩니다. 24 시간 후에는 파일을 더 이상 사용할 수 없습니다.

  {
    "input": "s3://zencodertesting/test.mov",
    "outputs": [
      {
        "url": "s3://my-output-bucket/final-1.mp4"
      },
      {
        "url": "s3://my-output-bucket/final-2.webm"
      }
    ]
  }

region

region:String

API 버전: V1, V2

기본값: us

유효한 값:미국, 유럽, 아시아, sa, 호주, 미국 버지니아, 미국-오레곤, 미국-캘리포니아, EU-더블린, 아시아-싱가포르, 아시아-도쿄, sa-saopaulo, 호주-시드니, us-중앙-gce, eu-west-gce 또는 아시아-동-GCE

호환 가능한 작업 유형: 모두

예:미국-버지니아

기술:

작업 인코딩에 사용할 Amazon AWS 리전을 지정할 수 있으며 지정된 리전의 서버에서 작업을 처리합니다.

대륙 전체 지역 (미국, 유럽, 아시아, sa 또는 오스트레일리아)을 지정할 수 있습니다. 이들 중 하나를 사용하면 대륙 내의 모든 지역을 사용하여 작업을 처리 할 수 있습니다.

보다 구체적인 리전은 us-virginia, us-oregon, us-n-california, eu-dublin, asia-singapore, asia-tokyo, sa-saopaulo 및 australia-sydney와 같은 특정 AWS 리전을 대상으로하는 데 사용할 수 있습니다.

  {
    "input": "s3://zencodertesting/test.mov",
    "region": "us-virginia"
  }

aspera_transfer_policy

aspera_transfer_policy:String

API 버전: V1, V2

기본값:페어

유효한 값:공정 또는 낮음

호환되는 작업 유형: vod

기술:

Aspera 서버에서 파일을 전송할 때 Zencoder가 인코딩 서버의 사용 가능한 대역폭을 사용하는 방법을 설정합니다.

Fair는 인코딩 서버에서 실행 중인 모든 작업 간에 대역폭을 균등하게 분할합니다.

Low는 인코딩 서버에서 사용 가능한 대역폭을 사용하지만 다른 작업에 우선 순위를 부여합니다. 낮음은 너무 많은 트래픽으로 서버를 과도하게 실행하는 것이 염려되는 경우 유용합니다.

참고: Aspera의 높음 옵션은 Zencoder에서 지원되지 않습니다. 기본적으로 가능한 한 많은 대역폭을 사용하지만 인코딩 서버의 다른 모든 작업보다 한 작업에 대한 대역폭 액세스의 우선 순위를 지정하지 않습니다.

  {
    "input": "s3://zencodertesting/test.mov",
    "aspera_transfer_policy": "fair"
  }

참고 항목: transfer_minimum_rate및transfer_maximum_rate

transfer_minimum_rate

transfer_minimum_rate:Number

API 버전: V1, V2

기본: 1000

유효한 값: 1000-249999

호환되는 작업 유형: vod

기술:

Aspera로 파일을 전송하는 동안 사용할 전송률 하한선을 설정합니다. 이것은 속도에 도달하는 것을 보장하지는 않지만 대역폭을 사용할 수있는 경우 파일 전송이이 속도 이하로 떨어지지 않습니다.

이 값보다 작아야transfer_maximum_rate합니다.

  {
    "input": "s3://zencodertesting/test.mov",
    "transfer_minimum_rate": 2500
  }

참고 항목: transfer_maximum_rate및aspera_transfer_policy

transfer_maximum_rate

transfer_maximum_rate:String

API 버전: V1, V2

기본: 250000

유효한 값: 1001-250000

호환되는 작업 유형: vod

기술:

Aspera로 파일을 전송하는 동안 사용할 전송 속도 한도를 설정합니다. 이것은 속도에 도달한다는 것을 보장하지는 않지만 대역폭을 사용할 수있는 경우 파일 전송이이 속도를 초과하지 않습니다.

많은 수의 작업을 동시에 인코딩 할 때 최대 전송 속도를 설정하여 트래픽으로 소스 및 대상 서버에 부담을주지 않도록하는 것이 좋습니다.

이 값이 보다 커야transfer_minimum_rate합니다.

  {
    "input": "s3://zencodertesting/test.mov",
    "transfer_maximum_rate": 25000
  }

참고 항목: transfer_minimum_rate및aspera_transfer_policy
