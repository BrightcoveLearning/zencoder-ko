페이지 내용

    인코딩 설정

    이 항목에서는 모든 인코딩 설정 표와 각 설정에 대한 자세한 안내서에 대한 링크를 제공합니다.

    직무

    설정 기본값 설명
    api_key 없음 Zencoder 계정의 API 키입니다.
    dynamic_profile_options 없음 CAE를 사용하면 이러한 출력의 출력 수와 매개 변수가 각 입력 파일에 대해 동적으로 변경될 수 있습니다. Dynamic_profile_options 설정은 변환을 만드는 방법을 정의합니다.
    generate_dynamic_profile 거짓 CAE를 사용하면 이러한 출력의 출력 수와 매개 변수가 각 입력 파일에 대해 동적으로 변경될 수 있습니다.
    input 없음 우리가 코드 변환 파일을 다운로드 할 수있는 HTTP, S3, 클라우드 파일, GCS, FTP, FTPS, SFTP, 푸른, 또는 아스페라 URL.
    outputs 없음 출력 설정의 배열 또는 해시입니다.
    region 우리 파일이 처리되는 영역입니다. 미국, 유럽, 아시아 또는 호주
    test 거짓 작업에 대해 테스트 모드 (“통합 모드”) 를 활성화합니다.
    private 거짓 작업에 대한 개인 정보 모드를 활성화합니다.
    download_connections 5 다운로드 가속을 위해 여러 개의 동시 연결을 사용합니다 (경우에 따라).
    pass_through 없음 이 작업과 함께 저장할 선택적 정보입니다.
    mock 거짓 조롱 된 작업 요청을 보냅니다.
    grouping 없음 이 작업에 대한 보고서 그룹화
    aspera_transfer_policy 공정 Aspera 파일 전송에 사용 가능한 대역폭을 할당하는 방법.
    transfer_minimum_rate 1000 데이터 전송 최소의 목표 속도 (Kbps) 입니다.
    transfer_maximum_rate 250000 데이터 전송 최대값에 대한 목표 속도 (Kbps) 입니다.
    expected_md5_checksum 없음 입력 파일의 예상 체크섬입니다.
    credentials 없음 저장된 자격 증명을 별명으로 참조합니다.

    일반 출력 설정

    설정 기본값 설명
    type 표준 출력 할 파일의 유형입니다.
    label 없음 이 출력에 대한 선택적 레이블입니다.
    dynamic_profile_rendition 없음 dynamic_profile_rendition 설정은 이 출력에 동적으로 생성된 CAE 프로파일의 설정, 특히 설정을 사용할 프로파일의 변환을 사용하도록 지정합니다.
    url 없음 트랜스코딩된 파일을 넣을 S3, 클라우드 파일, GCS, FTP, FTPS, SFTP, 아스페라, 푸른, HTTP 또는 RTMP URL입니다.
    secondary_url 없음 우리는 트랜스코딩된 파일을 넣을 S3, 클라우드 파일, GCS, FTP, FTPS, SFTP, HTTP, RTMP, 아스페라 또는 푸른 URL.
    base_url 없음 파일 이름 없이 트랜스코딩된 파일을 넣을 기본 S3, 클라우드 파일, GCS, FTP, FTPS, SFTP, Azure 또는 아스페라 디렉터리 URL입니다.
    filename 없음 완성 된 파일의 파일 이름입니다.
    package_filename 없음 패키지 출력의 파일 이름입니다.
    package_format 없음 출력 파일에 사용할 ZIP/패키징 형식입니다.
    device_profile 없음 모바일 장치 호환성에 사용할 장치 프로필입니다.
    strict 거짓 엄격 모드를 활성화합니다.
    skip_video 거짓 비디오 트랙을 출력하지 마십시오.
    skip_audio 거짓 오디오 트랙을 출력하지 않습니다.
    source 없음 다른 작업의 레이블을 참조하고 해당 출력에서 만든 비디오를 입력 파일 대신 처리하기 위해 사용합니다.
    credentials 없음 저장된 자격 증명을 별명으로 참조합니다.
    generate_md5_checksum 거짓 출력 파일의 MD5 체크섬을 생성합니다.
    parallel_upload_limit S3의 경우 30, 다른 모든 목적지의 경우 10. 시도할 동시 업로드의 최대 수입니다.
    headers 없음 우리가 그것을 업로드 할 때 파일과 함께 보낼 HTTP 헤더.

    CAE 설정

    설정 기본값 설명
    min_renditions 1 생성할 최소 변환 수입니다.
    The maximum number of renditions to generate 8 생성할 최대 변환 수입니다.
    min_resolution 192x108 사다리에 대해 고려해야 할 최소 해상도입니다.
    max_resolution 소스 해상도 래더에 대해 고려해야 할 최대 해상도입니다.
    max_frame_rate 30 사용할 수 있는 최대 프레임 속도.
    keyframe_rate 0.5 키프레임 속도 이 값은 0.5 또는 1일 수 있습니다. SSAI가 활성화된 콘텐츠의 경우 이 값은 1이어야 합니다.
    fixed_frame_rate 진정한 생성기에 모든 변환에서 동일한 프레임 속도를 사용하도록 지시합니다. false인 경우 360p 이하의 해상도에서 H.264 기준선 프로파일 변환에 프레임 속도가 절반으로 할당됩니다.
    min_bitrate 200 사용할 수 있는 최소 비트 전송률 [kbps] 입니다.
    max_bitrate 15000 사용할 수 있는 최대 비트레이트 (kbps) 입니다.
    max_first_rendition_bitrate 400 첫 번째 변환에 사용할 수 있는 최대 비트 전송률 [kbps] 입니다.
    min_granularity 30 변환 간 비트 전송률의 최소 상대 변화.
    max_granularity 90 변환 간 비트 전송률의 최대 상대 변화입니다.
    video_codec h264 사용할 비디오 코덱.
    bitrate_cap_to_bitrate_ratio 1.3 VBR 인코딩에 사용할 최대 비트 전송률과 평균 비트 전송률의 비율입니다.
    buffer_size_to_bitrate_ratio 2.0 VBR 인코딩에 사용할 디코더 버퍼 크기와 평균 비트 전송률의 비율입니다.
    bitrate_cap_offset 64 최대 비트 전송률을 계산할 때 상수가 추가됩니다.
    buffer_size_offset 85 VBV 버퍼 크기를 계산할 때 상수가 추가됩니다.
    quality_rate_tune 0 품질 대 속도 조정 매개 변수.
    select_baseline_profile_configuration 진정한 프로파일에 사용된 변환이 하나 이상 기준선 프로파일이 됩니다.
    video_configurations 없음 CAE 인코딩의 일부로 우선적으로 또는 필수적으로 생성되어야 하는 해상도, 프레임 속도 및 관련 코덱 설정을 설명하는 구조의 배열입니다.

    형식 및 코덱

    설정 기본값 설명
    format 출력 파일 이름과 비디오 또는 오디오 코덱에 의해 결정됩니다. 그렇지 않으면: mp4 (표준 출력의 경우); ts (세그먼트 된 출력의 경우). 사용할 출력 형식입니다.
    video_codec 형식, 프로파일 또는 에 의해 결정됩니다audio_codec . 아무 것도 제공되지 않으면 h264. 사용할 비디오 코덱입니다.
    audio_codec 형식, 프로필 또는 에 따라 결정됩니다video_codec . aac (제공되지 않은 경우). 사용할 오디오 코덱입니다.

    해상도

    설정 기본값 설명
    size 없음 출력 비디오의 해상도 (wxH, 픽셀 단위) 입니다.
    width 없음 출력 비디오의 최대 너비 (픽셀 단위) 입니다.
    height 없음 출력 비디오의 최대 높이 (픽셀 단위) 입니다.
    upscale 거짓 입력이 대상 출력 해상도보다 작은 경우 출력을 업스케일합니다.
    aspect_mode 영역 입력 파일의 종횡비가 대상 너비/높이 종횡비와 일치하지 않을 때 수행 할 작업.
    sample_aspect_ratio 없음 출력 비디오에 사용할 샘플 (픽셀) 종횡비입니다. 크기, 너비, 높이 및aspect_mode모드 옵션은 모두 디스플레이 치수가 아닌 인코딩된 픽셀 치수를 나타냅니다.
    force_interlace_headers 거짓 일부 디코더는 인터레이스 콘텐츠를 요청하지만 프로그레시브 콘텐츠는 잘 재생됩니다. 이 설정은 프로그레시브 콘텐츠의 메타데이터에 “인터레이스” 헤더를 추가하여 실제로 인터레이스된 것처럼 보이게 합니다.

    속도 제어

    설정 기본값 설명

    video_bitrate 없음 대상 비디오 비트 전송률 (kbps) 입니다. 특정 비트 전송률을 목표로 삼지 않는 한 품질 설정을 선택하는 경우에는 필요하지 않습니다.
