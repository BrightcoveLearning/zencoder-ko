vp6_temporal_down_watermark

vp6_temporal_down_watermark:Number

API 버전: V1, V2

상위: 출력

기본: 20

유효한 값: 0-100

호환 가능한 작업 유형: 모두

예: 10

기술:

임시 다운 워터 마크 백분율에 대한 코덱 매개 변수입니다. 인코더가 프레임 삭제를 시작할 수있는 나머지 데이터 속도 버퍼의 백분율을 지정합니다. vp6_temporal_리샘플링이 활성화된 경우에만 사용됩니다. 비율이 클수록 요청 된 비디오 비트 전송률을 달성하기 위해 프레임이 삭제 될 가능성이 높아집니다. 그 반대도 사실입니다.

{
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "vp6_temporal_down_watermark": 10
    }
  ]
}

참조: vp6_임시 리샘플링

vp6_temporal_resampling

vp6_temporal_resampling:Boolean

API 버전: V1, V2

상위: 출력

기본: 픽셀 당 비트 수가 적은 인코딩의 경우 참입니다. 그렇지 않으면 거짓입니다.

유효한 값:참 또는 거짓

호환 가능한 작업 유형: 모두

예: true

기술:

시간적 리샘플링을위한 코덱 매개 변수입니다. 특히 어려운 지역에서 활성화되면 인코더는 목표 데이터 속도를 달성하기 위해 프레임을 삭제합니다. vp6_temporal_down_watermark의 값에 따라 인코더가 프레임 드롭을 시작할 수 있는 데이터 레이트 버퍼의 백분율이 결정됩니다.

낮은 픽셀 당 비트 수를 사용하는 파일에서는 기본값이 true이고 픽셀 당 비트 수가 많은 파일에서는 false입니다. 계산하려면비디오_비트 전송률을너비* 높이* frame_rate로 초당 비트 수로 나눕니다. . 이 숫자가 0.07 미만이면 vp6_temporal_resampling이 기본적으로 true로 설정됩니다. 예를 들면 다음과 같습니다. 초당 600,000 비트 (약 600kbps)를 (640x480 * 30fps) = 0.065로 나눈 값이므로이 경우 vp6_temporal_resampling이 켜집니다.

참고: constant_bitrate모드를 사용하는 경우 이 옵션은 true로 설정됩니다. vp6_임시 _다운_워터마크를조정하여 결과를 조정합니다.

{
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "vp6_temporal_resampling": true
    }
  ]
}

참조: vp6_임시 _다운_워터마크

vp6_compression_mode

vp6_compression_mode:String

API 버전: V1, V2

상위: 출력

기본값:좋음

유효한 값:양호 또는 최고

호환 가능한 작업 유형: 모두

예:베스트

기술:

출력 품질과 비교하여 인코딩 속도에 영향을줍니다. "좋은"은 속도와 품질의 균형을 유지합니다. "최상"은 속도보다 품질을 우선시하므로 압축률은 높아지지만 인코딩 속도는 느려집니다. 속도를 1 또는 2로 설정하고 VP6을 사용하는 경우 “최상의” 모드를 사용합니다. 그렇지 않으면 “good”를 사용합니다.

{
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "vp6_compression_mode": "best"
    }
  ]
}

참조: 속도

vp6_2pass_min_section

vp6_2pass_min_section:Integer

API 버전: V1, V2

상위: 출력

기본: 40

유효한 값: 0-100

호환 가능한 작업 유형: 모두

예: 50

기술:

2패스 VBR 인코딩의 경우, 이 기능은 섹션이 아무리 쉬우더라도 인코더가 모든 섹션에 허용할 수 있는 가장 낮은 데이터 레이트를 제어합니다. 이 값은 어려운 섹션이 쉬운 섹션에서 너무 많은 비트를 훔치는 것을 방지하는 데 사용됩니다. 백분율로 표시되며 정수로 표시됩니다.

예를 들어, 비디오_비트전송률이 1000kbps로 설정되고 vp6_2_pass_min_섹션이 50으로 설정된 경우 모든 섹션의 가장 낮은 비트 전송률은 500kbps (1000 * 50%) 입니다.

{
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "vp6_2pass_min_section": 50
    }
  ]
}

참조: vp6_2패스_맥스_섹션

vp6_denoise_level

vp6_denoise_level:Float

상위: 출력

기본: 0

유효한 값: 0과 1 사이의 소수

호환 가능한 작업 유형: 모두

예: 0.1

기술:

소스의 노이즈 레벨을 지정합니다. 0보다 큰 값은 소스 내에서 일정하고 알려진 수준의 노이즈를 의미합니다. 값이 0이면 필터가 노이즈 레벨을 추정하여 프레임마다 다를 수 있습니다. 노이즈 레벨이 클수록 노이즈 소스가 더 많아 필터에 의한 제거가 증가하지만 선명도는 떨어집니다.

{
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "vp6_denoise_level": 0.1
    }
  ]
}

참조: 노이즈 제거

상수 _ 비트 레이트

constant_bitrate:Boolean

API 버전: V1, V2

상위: 출력

기본값:거짓

유효한 값:참 또는 거짓

호환 가능한 작업 유형: 모두

예: true

기술:

CBR (고정 비트 레이트) 인코딩은 일부 스트리밍 서버 및 디코더에서 사용됩니다. CBR 인코딩은 동일한 비트 전송률에서 VBR (가변 비트 전송률) 인코딩보다 훨씬 더 나빠 보이므로 반드시 필요한 경우에만이 설정을 사용하십시오. 이 설정은 다음을 설정해야 합니다. video_bitrate ; 당신은 사용할 수 없습니다품질 constant_bitrate 설정을 사용할 때 비트 전송률을 자동 선택합니다.

HTTP 기반 전송 또는 다운로드 된 비디오에 constant_bitrate를 사용하지 마십시오.

다른 video_codecs는 constant_bitrate 설정에 따라 다르게 작동합니다.

{
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "constant_bitrate": true
    }
  ]
}
