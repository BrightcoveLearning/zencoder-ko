개요

Zencoder가 지원하는 HDR 형식과 HDR 형식 매개 변수에 대한 권장 사항에 대한 자세한 내용은 HDR 비디오 가이드를 참조하십시오 .

참고: HDR 출력은 HEVC(H.265) 소스에서만 생성할 수 있습니다. H.264 소스의 HDR 출력은 지원되지 않습니다.

preserve_hdr_metadata

master_display:Boolean

API 버전: V2

상위: 출력

기본값:거짓

호환되는 작업 유형: vod

기술:

true로 설정하면 이 매개 변수는 Zencoder가 입력 파일에서 HDR 관련 메타데이터를 가져오고 출력으로 전달하도록 지시합니다.

예를 들어 다음 작업 요청을 사용하여 HDR10, PQ10 또는 HLG10 형식의 HEVC 인코딩 비디오 스트림 트랜스 코딩을 수행 할 수 있습니다. 

{
  "input": "s3://mybucket/hdr_master.mp4",
  "outputs": [
    {
     "video_codec": "hevc",
     "video_codec_profile": "main10",
     "video_bit_depth": 10,
     "preserve_hdr_metadata": true
    }
  ]
}

이 매개 변수는 HDR 관련 메타 데이터를 직접 지정하여 다른 매개 변수와 함께 사용할 수도 있습니다. 예를 들어, JSON 작업 요청 아래에는preserve_hdr_metadata매개 변수가 이월하는 데 사용되는 방법이 나와 있습니다. master display HEVC로 인코딩된 돌비비전 8.1 스트림의content light level메타데이터 및 DolbyVision동적 메타데이터 ( RPU파일) 는 dolby_vision_rpu_url 매개 변수를 사용하여 별도로 전달됩니다.

{
  "input": "s3://mybucket/dv81_master.mp4",
  "outputs": [
    {
     "video_codec": "hevc",
     "video_codec_profile": "main10",
     "video_bit_depth": 10,
     "hdr_format": "dv8.1",
     "preserve_hdr_metadata": true,
     "dolby_vision_rpu_url": "s3://mubucket/dv81_master_rpu.rpu"
    }
  ]
}

hdr_format

hdr_format:String

API 버전: V2

부모: outputs , dynamic_profile_options

유효한 값: hdr10 , pq , hlg , dv5 , dv8.1

기본값:없음

호환되는 작업 유형: vod

기술:

출력을 생성하는 데 사용되는 HDR 형식의 유형을 지정합니다.

이 매개 변수는 입력 파일에 HDR 형식 관련 메타 데이터가 없기 때문에 형식 유형과 모든 관련 메타 데이터를 직접 지정해야하는 경우에 사용해야합니다.

예를 들어, J2K 인코딩 MFX로 제공되는 PQ 색상 등급 콘텐츠는 다음 작업 요청을 통해 PQ10 출력 스트림으로 변환 할 수 있습니다.

{
  "input": "s3://mybucket/j2k_master.mfx",
  "outputs": [
    {
     "video_codec": "hevc",
     "video_codec_profile": "main10",
     "video_bit_depth": 10,
     "hdr_format": "pq"
    }
   ]
}

또는. 동일한 J2K MXF 마스터가 HDR10 콘텐츠를 나타내며마스터 디스플레이및콘텐츠 라이트 레벨메타데이터 매개 변수를 설명하는 추가 파일이 함께 제공되는 경우 HDR10 HEVC에서 인코딩스트림은 다음 작업 요청을 사용하여 수행할 수 있습니다.

{
  "input": "s3://mybucket/j2k_master.mfx",
  "outputs": [
    {
      "video_codec": "hevc",
      "video_codec_profile": "main10",
      "video_bit_depth": 10,
      "hdr_format": "hdr10",
      "master_display": {
        "red": {"x": 0.708, "y": 0.292},
        "green": {"x": 0.17, "y": 0.797},
        "blue": {"x": 0.131, "y": 0.046},
        "white_point": {"x": 0.31273,"y": 0.329},
        "luminance": {"max": 1000,"min": 0.0001}
      },
      "max_content_light_level": 1000,
      "max_frame_average_light_level": 180
    }
  ]
}

video_color_range

video_color_range:String

API 버전: V2

부모: outputs , dynamic_profile_options

유효한 값: limited , full

기본값:없음

호환되는 작업 유형: vod

기술:

픽셀 값의 동적 범위를 나타냅니다. 예를 들어 8비트 픽셀의 경우limited Y(luma) 픽셀 값이 16에서 235 사이에 유지되는 반면full Y(luma) 픽셀 값이 0에서 255 사이의 범위에 있음을 의미합니다.

대부분의 콘텐츠는 현재limited픽셀을 사용하여 제작 및 인코딩됩니다. 그러나full -range 픽셀도 때때로 사용될 수 있습니다. 예를 들어, 10 비트 폭의 전체 범위 픽셀은 DolbyVision 5 형식으로 사용됩니다.

사용 예: 

{
  "input": "s3://mybucket/dv5_master.mp4",
  "outputs": [
   {
    "video_codec": "hevc",
    "video_codec_profile": "main10",
    "video_bit_depth": 10,
    "hdr_format": "dv5",
    "video_color_range": "full",
    "dolby_vision_rpu_url": "s3://mubucket/dv5_master_rpu.rpu"
   }
  ]
}
