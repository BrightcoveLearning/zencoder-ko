워터 마크

워터마크:배열 또는 해시

API 버전: V1, V2

상위: 출력OR 다이나믹_프로파일_옵션

유효한 값: 워터 마크 설정의 배열 또는 해시

호환 가능한 작업 유형: 모두

기술:

워터 마킹 API를 사용하여 출력 비디오에 하나 이상의 워터 마크를 추가 할 수 있습니다. Zencoder는 JPEG, BMP 또는 PNG 형식의 출력 당 최대 4 개의 워터 마크 파일을 지원합니다.

  • GIF, 32 비트 RGB + 알파 및 64 비트 RGB + 알파 워터 마크 이미지는 지원되지 않습니다.
  • 비디오 가장자리를 넘어 확장되는 워터 마크는 적용되지 않습니다. 이 경우 출력에 경고가 발생합니다.
  • VP6 코덱을 사용할 때 몇 가지 추가 제한 사항이 있습니다. 비디오에 하나의 워터 마크 만 적용 할 수 있습니다 (여러 워터 마크를 통과 한 경우 첫 번째 워터 마크 만 사용됨). 또한 워터 마크 크기 조정은 지원되지 않으며 워터 마크 이미지는 PNG 파일이어야합니다.
  { "input": "s3://zencodertesting/test.mov", "outputs": [ { "watermarks": [ { "url": "s3://bucket/watermark_file.png", "x": "20", "y": "-10%", "width": 32, "height": 24 } ]}]}

url

URL:문자열

API 버전: V1, V2

상위: 출력/워터마크

유효한 값: 이미지 파일에 대한 유효한 URL입니다.

호환 가능한 작업 유형: 모두

예:

  • https://s3.amazonaws.com/bucket/img.png
  • ftp://user:pass@example.com/path/to/watermark.jpg

기술:

워터 마크로 사용할 원격 이미지 파일의 URL입니다. 입력 URL 구문을 사용합니다. 인증 유무에 관계없이 S3, GCS, 클라우드 파일, HTTP / S, FTP 및 SFTP를 지원합니다.

Zencoder는 GIF, JPEG, BMP 또는 PNG 형식의 워터 마크 파일을 지원합니다.

투명한 워터 마크의 경우 알파 또는 인덱스 투명도가있는 PNG를 사용합니다.

지원되는 PNG 형식

  • 8 비트 RGB
  • 8 비트 RGB + 알파
  • 8 비트 그레이 스케일
  • 8 비트 인덱스
  • 8 비트 인덱스 + 투명
  • 16 비트 RGB
  • 16 비트 그레이 스케일

지원되지 않는 PNG 형식

  • 8 비트 그레이 스케일 + 알파
  • 16 비트 그레이 스케일 + 알파
  • 16 비트 RGB + 알파
  { "input": "s3://zencodertesting/test.mov", "outputs": [ { "watermarks": { "url": "https://s3.amazonaws.com/bucket/img.png" } } } }

엑스

x:문자열 또는 숫자

API 버전: V1, V2

상위: 출력/워터마크

기본: -10

호환 가능한 작업 유형: 모두

예:

  • -0
  • 320
  • 중심

기술:

비디오 내 x 축 (왼쪽 / 오른쪽)에서 워터 마크를 배치 할 위치입니다. 픽셀 수 (예: 100 또는 -20), 비디오 너비의 백분율 (예: 25% 또는 -5%) 또는' '가운데로 배치하여 워터마크를 비디오의 가운데에 맞출 수 있습니다. 양수를 사용하여 비디오의 왼쪽을 기준으로 배치하고 음수를 사용하여 비디오의 오른쪽을 기준으로 배치합니다. "-0"(문자열)을 사용하여 오른쪽으로 잠급니다.

기본적으로 위치는 패딩을 포함하지 않고 보이는 콘텐츠 영역을 기반으로합니다. 원점을사용하여 전체 출력을 기반으로 워터마크 위치를 설정합니다.

  { "input": "s3://zencodertesting/test.mov", "outputs": [ { "watermarks": { "x": 320 } } } }

참조: y및원점

유래

원점: 문자열

API 버전: V1, V2

상위: 출력/워터마크

기본값:콘텐츠

유효한 값:콘텐츠 또는 프레임

호환 가능한 작업 유형: 모두

예:프레임

기술:

워터 마크 위치의 기준이되는 비디오 부분입니다. 이것은 aspect_mode가 'pad'인 작업에만 영향을 미칩니다.

  • 'frame은 패딩을 포함하여 출력의 전체 해상도를' 기반으로 배치를 기반으로 합니다.
  • '콘텐츠는 패딩을 포함하지 않고 표시되는 콘텐츠 영역에 배치를' 기반으로 합니다.
  { "input": "s3://zencodertesting/test.mov", "outputs": [ { "watermarks": { "origin": "content" } } } }

또한보십시오: NS , 와이 , 그리고 aspect_mode

API 버전: V1, V2

상위: 출력/워터마크

기본: 1.0

유효한 값: 0.0에서 1.0

호환 가능한 작업 유형: 모두

기술:
