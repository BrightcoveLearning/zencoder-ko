썸네일

축소판:배열 또는 해시

API 버전: V1, V2

상위: 출력

호환 가능한 작업 유형: 모두

예:

  • “축소판”: [ {} ]
  • "썸네일": {}

기술:

모든 출력에 대해 하나 이상의 축소판을 요청할 수 있습니다.

각 출력에는 고유한크기 , base_url 및access_control 세트가 있는 고유한 썸네일세트를 가질 수 있습니다.

배열 내에서 여러 축소판 해시를 전달하여 단일 출력에 대해 여러 축소판 집합을 만들 수 있습니다. 배열에 축소판 옵션이 전달되는 경우레이블을 set에 대해설정해야 합니다.

세트 S3 액세스 제어출력 비디오에 사용하는 것과 동일한 옵션을 사용합니다.

썸네일은 세트당 100 개로 제한됩니다. 요청 된 옵션에 따라 더 많은 썸네일이 생성되면 출력에 경고가 추가되고 한계에 도달하면 썸네일 생성이 중지됩니다.

참고 : 썸네일 전용 작업 같은 것은 없습니다. 썸네일로 출력 지정하기 또한 만들 것입니다 기본 트랜스코딩 설정으로 청구 가능한 전체 길이 mp4를 임시 저장소에 업로드합니다.

{ "input": "s3://zencodertesting/test.mov", "outputs": [ { "thumbnails": [ { "label": "first", "number": 10 } , { "label": "second", "interval_in_frames": 5000 } ]}]}

start_at_first_frame

시작_AT_FIRST_프레임:부울

API 버전: V1, V2

상위: 출력/축소판

기본값:거짓

유효한 값:참 또는 거짓

호환 가능한 작업 유형: 모두

예: true

기술:

축소판 생성에숫자옵션을 사용하면 첫 번째 프레임에서 축소판을 찍기 시작합니다.

예를 들어 동영상의 길이가 5초이고숫자옵션을 사용하여 5개의 미리보기 이미지를 요청하는 경우 0s, 1s, 2s, 3s 및 4초로 축소판이 생성됩니다. 일반적으로 0.83s, 1.66s, 2.49s, 3.32s 및 4.15s에서 축소판을 가져옵니다.

{ "input": "s3://zencodertesting/test.mov", "outputs": [ { "thumbnails": { "number": 5, "start_at_first_frame": true } } } }

API 버전: V1, V2

{ "input": "s3://zencodertesting/test.mov", "outputs": [ { "thumbnails": { "times": [0, 30, 60, 90] } } } }

aspect_mode

아스펙트_모드:문자열

API 버전: V1, V2

상위: 출력/축소판

기본값:보존

유효한 값:보존, 늘이기, 자르기 또는 패드

호환 가능한 작업 유형: 모두

예:패드

기술:

축소판을 만들 때 사용할 종횡비 모드입니다.

  • 보존 : 기본적으로 Zencoder는 출력 비디오 파일의 종횡비를 유지하므로 와이드 스크린 콘텐츠를 제출하고 표준 해상도를 요청하면 축소판 파일이 와이드 스크린 종횡비를 유지하고 요청 된 크기에 맞을 것입니다.
  • 스트레치 : 종횡비 모드가 "늘이기"이면 미리보기 이미지가 이미지를 왜곡하더라도 요청 된 너비 및 높이와 정확히 일치합니다.
  • 자르기 : 이 옵션은 Zencoder가 위 / 아래 또는 왼쪽 / 오른쪽에서 픽셀을 잘라 요청한 크기와 일치하도록 축소판을 "확대"하도록 지시합니다.
  • 패드 : 패드 옵션은 요청 된 프레임 크기와 일치하도록 썸네일을 레터 박스로 지정하도록 Zencoder에 지시합니다. 이 옵션을 사용하여 출력 비디오의 종횡비를 유지하지만 항상 동일한 크기의 축소판을 가져옵니다.

참고: 비디오 스케일링 알고리즘의 제약으로 인해 썸네일의 크기는 짝수입니다. 예를 들어 100x75 크기의 미리보기 이미지를 요청하면 100x74 크기의 미리보기 이미지가 표시됩니다.

{ "input": "s3://zencodertesting/test.mov", "outputs": [ { "thumbnails": { "aspect_mode": "pad", "width": 200, "height": 200 } } } }

또한보십시오:크기 , 너비 , 그리고키

크기

크기: 문자열

API 버전: V1, V2

상위: 출력/축소판

호환 가능한 작업 유형: 모두

예: 400x300

기술:

"160 × 120"과 같은 썸네일의 목표 해상도. 크기가 제공되지 않으면 썸네일은 출력 비디오와 동일한 크기가됩니다. 기본적으로 가로 세로 비율을 유지하므로이 크기 매개 변수의 가로 세로 비율이 영화의 가로 세로 비율과 일치하지 않으면 결과 파일이이 크기와 정확히 일치하지 않을 수 있습니다. 해당 출력 파일에 대해 Stretch가 true로 설정되어 있으면이 축소판도 해상도를 채우기 위해 늘어납니다.

{ "input": "s3://zencodertesting/test.mov", "outputs": [ { "thumbnails": { "size": "400x300" } } } }

base_url

베이스_URL:문자열

API 버전: V1, V2

상위: 출력/축소판

유효한 값: 유효한 S3, 클라우드 파일, GCS, FTP, FTPS 또는 SFTP URL (파일 이름 제외). 형식: s3://[버킷 이름]/[경로] 또는 ftp://[user]:[password]@[ftp.url]/[path]

호환 가능한 작업 유형: 모두

예:

  • s3: //내 출력 버킷/
  • cf: //사용자명:api_key @container /경로/
  • ftp://user:password@ftp.example.com/path/to

기술:

축소판의 출력 대상입니다. base_url이 비어 있으면 Zencoder S3 버킷에 축소판을 저장합니다. Zencoder S3 버킷에 저장된 파일은 24 시간 후에 사용할 수 있습니다. 파일 이름은 출력 (예: frame_0000.png) 간에 고유하지 않으므로 파일을 덮어쓰지 않도록 대상은 고유 디렉토리 또는 키 접두사여야 합니다.

FTP 사용자 : 서버의 루트에서 쓰려고 시도하므로 URL의 절대 경로를 사용하여 서버에 성공적으로 쓸 수 있습니다.

자격 증명 참고 사항:

base_url을 통해 자격 증명을 제공하면작업 세부 정보 , 출력 세부정보 등에 대한 요청에 나타나는 URL에 표시됩니다. 자격증명이 Zencoder와 함께 저장되면 해당 응답의 URL에 표시되지 않습니다.

{ "input": "s3://zencodertesting/test.mov", "outputs": [ { "thumbnails": { "base_url": "s3://my-output-bucket/" } } } }

참조: 접두사및파일 이름

접두사

접두사:문자열

API 버전: V1, V2

상위: 출력/축소판

기본값:프레임

호환 가능한 작업 유형: 모두

예:

  • 엄지 손가락
  • video_thumbnails

기술:

썸네일 파일은 순차적 인 파일 이름으로 내보내집니다. 그러나 축소판 파일에 대해 사용자 지정 접두사를 지정할 수 있습니다. 접두사가 지정되지 않은 경우 프레임을 사용합니다. 예를 들어 접두사 'custom'을 사용하는 경우 내 보낸 파일의 이름은 custom_0000.png, custom_0001.png 등이됩니다. 사용자 지정 접두사가 없으면 이름이 frame_0000.png, frame_0001.png 등으로 지정됩니다.

{ "input": "s3://zencodertesting/test.mov", "outputs": [ { "thumbnails": { "prefix": "thumbs" } } } }

참조: base_url및파일 이름

파일 이름

파일 이름:문자열

API 버전: V1, V2

상위: 출력/축소판

기본값:프레임

호환 가능한 작업 유형: 모두

예:

{% raw %}
  • 썸네일_ {{number}}
  • {{number}} _ {{width}} x {{height}} -썸네일

기술:

{% endraw %}

썸네일 파일은 여러 속성에 따라 파일 이름을 지정할 수 있습니다. 속성에는 숫자, 패딩 된 숫자, 너비, 높이 및 크기가 포함됩니다. 파일 확장자는 "형식"옵션에 따라 Zencoder에 의해 추가되므로 추가 할 필요가 없습니다.

숫자 또는 패딩 된 숫자를 사용해야하며 나머지는 모두 선택 사항입니다. padded-number는 4 개의 스팟 (0000, 0001 등)을 사용합니다.

높이, 너비 및 크기는 생성 된 미리보기 이미지의 실제 크기를 기반으로합니다.

예: {{number}} _ {{width}} x {{height}} -썸네일

유효한 문자에는 문자, 숫자, 대시, 밑줄 및 보간 된 값이 포함됩니다.

URL로 인코딩된 경우에도 특수 문자는 파일 이름에서 지원되지 않습니다.

참고 : 파일 이름을 지정하면 지정된접두사가무시됩니다.

{“입력”: “s3: //zencodertesting/test.mov”, “출력”: [{“축소판 그림”: {“파일 이름”: " {{number}} _ {{width}} x {{height}} -썸네일”}}}}

헤더

헤더:해시

API 버전: V1, V2

상위: 출력 OR 축소판

호환 가능한 작업 유형: 모두

예: { "Content-Type": "binary/octet-stream" }

기술:

업로드 할 때 미리보기 이미지와 함께 보낼 HTTP 헤더입니다. 이 기능은 현재 S3 및 Cloud Files를 사용할 때 지원됩니다.

Zencoder는 다음 헤더의 제한된 하위 집합 설정을 지원합니다. 캐시 제어, 콘텐츠 처리, 콘텐츠 인코딩, 콘텐츠 유형, 만료, x-amz-acl, x-amz-스토리지 클래스, x-amz-서버 측 암호화 및 x-amz-meta-*.

출력 위치가 지정되지 않으면 헤더가 무시됩니다.

{ "input": "s3://zencodertesting/test.mov", "outputs": [ { "thumbnails": { "headers": { "Content-Type": "binary/octet-stream", "x-amz-acl": "public-read-write" } }}]}

예: ftp_dev_server

기술:

전송에 사용할 저장된 자격 증명의 닉네임을 참조합니다. 이 닉네임은계정 자격 증명 섹션에서관리됩니다.

노트: 신디케이션 서비스에 대한 자격 증명은 출력 비디오 전용입니다.

{ "input": "s3://zencodertesting/test.mov", "outputs": [ { "thumbnails": { "base_url": "ftp://ftp.example.com/thumbnails/", "credentials": "ftp_dev_server" } } } }

참조: base_url

parallel_upload_limit

패러렐_업로드_제한:정수

API 버전: V1, V2

기본: S3의 경우 30, 기타 모든 대상의 경우 10.

유효한 값: 1에서 30 사이의 정수

호환 가능한 작업 유형: 모두

예: 5

기술:

썸네일을 업로드 할 때 생성되는 최대 동시 업로드 수입니다.

원격 서버의 대역폭에 따라 전송 시간이 빨라질 수 있습니다. 더 많은 연결이 서버에 더 많은 부하를 줄 수 있습니다. 업로드 시간 초과에 문제가 있거나 Zencoder가 파일을 업로드 할 때 너무 많은 대역폭을 사용하지 않도록하려면이 값을 1로 설정하십시오.

{ "input": "s3://zencodertesting/test.mov", "outputs": [ { "thumbnails": { "base_url": "ftp://ftp.example.com/thumbnails/", "parallel_upload_limit": 5 } } } }
