개요

Zencoder는 "미사용 암호화"패러다임을 사용하여 암호화 된 미디어 파일 처리를 지원합니다. 당사는 귀하의 서버에서 암호화 된 파일을 검색하고 처리하는 동안에 만 해독합니다. 해독 된 파일 버전은 처리 후 즉시 제거되어 데이터를 안전하게 유지합니다.

CBC 및 CTR 모드에서 AES-128 및 AES-256 암호화 파일을 모두 지원합니다. 복호화 매개 변수는 작업 요청에 포함되어야합니다. 암호 해독은 암호해독_키또는 암호해독_key_url이 지정되면 활성화됩니다.

decryption_method

decryption_method:String

API 버전: V1, V2

부모: 입력

기본값: aes-128-cbc ( 암호 해독 키또는 암호해독_key_url이 설정된 경우)

유효한 값:“없음”, “AES-128-cbc”, “AES-128-ctr”, “애스-256-cbc” 또는 “애스-256-ctr”

호환되는 작업 유형: vod

예: aes-128-cbc

기술:

복호화에 사용할 복호화 알고리즘을 설정합니다.

  {
    "input": "s3://zencodertesting/test.mov",
    "decryption_method": "aes-128-cbc"
  }

암호 해독 키

decryption_key:String

API 버전: V1, V2

부모: 입력

유효한 값: 16진수 문자열 16진수 (32자 길이, 선택적 “0x” 접두사)

호환되는 작업 유형: vod

예: 1234567890abcdef1234567890abcdef

기술:

암호화 된 입력에 사용할 암호 해독 키를 설정합니다.

  {
    "input": "s3://zencodertesting/test.mov",
    "decryption_key": "1234567890abcdef1234567890abcdef"
  }

암호 해독_key_url

decryption_key_url:String

API 버전: V1, V2

부모: 입력

유효한 값: 16 바이트 바이너리 키를 포함하는 파일에 대한 URL

호환되는 작업 유형: vod

예: http://example.com/encryption.key

기술:

이진 형식의 키 (총 16 바이트)와 함께 사용할 암호 해독 키 파일의 URL입니다. 암호 해독_key가 지정되면 무시됩니다. 
  {
    "input": "s3://zencodertesting/test.mov",
    "decryption_key_url": "http://example.com/encryption.key"
  }

암호 해독

decryption_password:String

API 버전: V1, V2

부모: 입력

유효한 값: 문자열

호환되는 작업 유형: vod

예:슈퍼 비밀번호-암호

기술:

입력 파일을 해독하기 위해 키와 함께 사용되는 암호입니다.

  {
    "input": "s3://zencodertesting/test.mov",
    "decryption_password": "super-secret-password"
  }
